## Phase 1 — Slug Fix

**Current (buggy):**
```bash
SLUG=$(echo "{url}" | sed 's|https\?://||;s|/.*||;s|\.|--|g')
```

**Fixed:**
```bash
SLUG=$(echo "{url}" | sed 's|https\?://||;s|/.*||;s|\.|-|g')
```

`s|\.|--|g` replaces each dot with two dashes, producing `example--com`. The fix `s|\.|-|g` produces the correct `example-com`. Every downstream path (output folder, GitHub tech pack URL, smoke test filename) inherits this slug.

## Phase 3 — Fingerprinting Additions

Six new detection blocks added to the Phase 3 section in SKILL.md, ordered by signal strength. Each block follows the existing pattern: HTTP headers first, then HTML source, then cookies/globals, then version extraction.

### Shopify

```bash
# Headers (Definitive)
curl -sI {url} | grep -i 'x-shopify-stage'

# HTML (Definitive)
curl -s {url} | grep -o 'cdn\.shopify\.com'
curl -s {url} | grep 'window\.Shopify\s*='

# Cookies (Definitive)
curl -sI {url} | grep -i 'set-cookie' | grep '_shopify_'
```

Version: `window.Shopify.theme.name` via JS eval in Phase 11 or inline script grep.

Log: `Framework: Shopify (source: x-shopify-stage header + cdn.shopify.com, confidence: definitive)`

### Strapi

```bash
# Headers (Definitive)
curl -sI {url} | grep -i 'x-powered-by: strapi'
# v5: X-Strapi-Version header present
curl -sI {url} | grep -i 'x-strapi-version'

# Endpoint probe (Definitive)
curl -s {url}/admin/init | python3 -c "
import sys, json
try:
    d = json.load(sys.stdin)
    print('strapi' if 'hasAdmin' in d.get('data', {}) else '')
except: pass
"

# Health check (High)
curl -s {url}/_health
```

Version: `X-Strapi-Version` header (v5+); otherwise probe `/admin/init` for version hints.

Log: `Framework: Strapi 5.x (source: X-Strapi-Version header + /admin/init, confidence: definitive)`

### Rails

```bash
# HTML (Definitive)
curl -s {url} | grep -i 'csrf-token'
curl -s {url} | grep -i 'authenticity_token'

# Headers (High)
curl -sI {url} | grep -i 'x-runtime'

# Cookies (Definitive)
curl -sI {url} | grep -oP '_[a-z0-9_]+_session'

# Rails 7+ importmap (High)
curl -s {url} | grep '@hotwired'
```

Version: extract `@hotwired/turbo@X.x` from importmap block; no direct Rails version in headers.

Log: `Framework: Rails 7.x (source: csrf-token meta + _*_session cookie, confidence: definitive)`

### Astro

```bash
# Generator meta (Definitive + version)
curl -s {url} | grep -o 'content="Astro v[^"]*"'

# Asset paths (High)
curl -s {url} | grep '/_astro/'

# Astro islands (Definitive)
curl -s {url} | grep 'astro-island'

# View Transitions (confirms Astro 3+)
curl -s {url} | grep 'data-astro-transition'
```

Version: extracted directly from `<meta name="generator" content="Astro vX.Y.Z">`. If suppressed, log `[TECH-DETECTED:astro:unknown]` with signals noted.

Log: `Framework: Astro 5.6.1 (source: generator meta, confidence: definitive)`

### Django

```bash
# HTML (Definitive)
curl -s {url} | grep -i 'csrfmiddlewaretoken'

# Admin panel (Definitive)
curl -s {url}/admin/ | grep -i 'django site administration'

# DRF browsable API (Definitive — API-only Django)
curl -s {url}/api/?format=api | grep -i 'django rest framework'

# Cookies (Definitive)
curl -sI {url} | grep -i 'set-cookie' | grep 'csrftoken'
```

Version: not exposed in production. Debug error page leaks it but `DEBUG=True` never in production.

Log: `Framework: Django (source: csrfmiddlewaretoken in HTML, confidence: definitive)`

### FastAPI

Two or more signals required — individual signals are High, not Definitive (docs/openapi can be disabled).

```bash
# Swagger UI (High — may be disabled)
curl -s {url}/docs | grep -i 'swagger-ui'

# OpenAPI spec (High — may be disabled)
curl -s {url}/openapi.json | python3 -c "
import sys, json
try:
    d = json.load(sys.stdin)
    print('fastapi' if 'openapi' in d else '')
except: pass
"

# uvicorn server header (High — combined)
curl -sI {url} | grep -i 'server: uvicorn'
```

Version: not exposed in production. `/openapi.json` `.info.version` is the app version, not FastAPI's.

Log: `Framework: FastAPI (source: uvicorn header + /docs swagger-ui, confidence: high-combined)`

## validate-fingerprinting.sh Design

**21 checks** (3 slug + 6 per-pack signal + 11 coverage loop + 1 count). Coverage loop count grows automatically as tech packs are added.

### Slug checks (3)

1. SKILL.md exists
2. Slug sed uses single dash — `s|\.|-|g` present in SKILL.md AND `s|\.|--|g` absent
3. Slug command produces correct output — inline bash test:
   ```bash
   result=$(echo "example.com" | sed 's|https\?://||;s|/.*||;s|\.|-|g')
   [ "$result" = "example-com" ]
   ```

### Per-tech-pack signal checks (6)

4. Astro signal present — grep SKILL.md for `_astro/` or `astro-island`
5. Django signal present — grep SKILL.md for `csrfmiddlewaretoken`
6. FastAPI signal present — grep SKILL.md for `uvicorn` or `swagger-ui`
7. Rails signal present — grep SKILL.md for `X-Runtime` or `csrf-token`
8. Shopify signal present — grep SKILL.md for `x-shopify-stage` or `cdn.shopify.com`
9. Strapi signal present — grep SKILL.md for `admin/init` or `x-strapi-version`

### Coverage loop (12)

10–20. Dynamic loop over `plugins/beacon/technologies/*/` — one check per directory:
   - For each directory name, grep SKILL.md (case-insensitive) for that framework name
   - Skip `graphql` explicitly — detected in Phase 6 (GraphQL introspection), not Phase 3
   - FAIL if any non-excluded tech pack directory has no mention in SKILL.md
   - Currently 11 iterations (12 packs minus graphql); grows automatically as packs are added

21. Tech pack directory count check — assert directory count equals expected (currently 12). Catches silent additions before the loop runs.

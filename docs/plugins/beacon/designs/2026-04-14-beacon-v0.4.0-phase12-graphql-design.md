## output-synthesis.md Design

### What Phase 12 Reads

| Session brief section | Used for |
|----------------------|----------|
| Infrastructure table | `tech-stack.md` rows + INDEX.md tokens |
| Discovered URLs list | `site-map.md` |
| Discovered Endpoints table (Method, Path, Auth, Phase, Notes) | `smoke-test.sh` check() calls |
| JS globals + API response constants | `constants.md` rows |
| Phase 11 results block | `constants.md` rows (enums, flags from JS globals) |
| `[PHASE-11-SKIPPED]` / `[OPENAPI-SKIPPED:...]` signals | `OPENAPI_STATUS` token resolution |

### Output Files

**`tech-stack.md`** — Copy Infrastructure table rows from session brief verbatim.

**`site-map.md`** — List every URL discovered across all phases, grouped by phase (Phase 2 crawl, Phase 5 sitemap, Phase 11 browsed).

**`constants.md`** — Populate the 6-section template from JS globals and API response values. Omit any section with no data. Flag nonces as session-specific.

**`scripts/test-{site-slug}.sh`** — One `check()` call per row in Discovered Endpoints table. Auth = No rows pass `"false"` as the fourth argument (active check). Auth = Yes rows pass `"true"` — the `check()` function prints a yellow SKIP line and increments `$SKIP`. `{{SMOKE_TEST_CHECKS}}` resolves to the full block of calls.

## smoke-test.sh.template Design

```bash
#!/usr/bin/env bash
# Smoke tests for {{SITE_SLUG}} — generated by beacon v{{PLUGIN_VERSION}}
# Usage: bash scripts/test-{{SITE_SLUG}}.sh [BASE_URL]
# Generated: {{DATE}}

BASE_URL="${1:-{{BASE_URL}}}"
PASS=0; FAIL=0; SKIP=0
GREEN='\033[0;32m'; RED='\033[0;31m'; YELLOW='\033[1;33m'; NC='\033[0m'

check() {
  local desc="$1" url="$2" expected="$3" auth_required="$4"
  if [ "$auth_required" = "true" ]; then
    echo -e "  ${YELLOW}SKIP${NC}  $desc (auth required)"
    SKIP=$((SKIP + 1)); return
  fi
  actual=$(curl -s -o /dev/null -w "%{http_code}" --max-time 10 "$url")
  if [ "$actual" = "$expected" ]; then
    echo -e "  ${GREEN}PASS${NC}  $desc → $actual"
    PASS=$((PASS + 1))
  else
    echo -e "  ${RED}FAIL${NC}  $desc → expected $expected, got $actual"
    FAIL=$((FAIL + 1))
  fi
}

echo "Smoke tests: $BASE_URL"
echo "========================================"

{{SMOKE_TEST_CHECKS}}

echo "========================================"
echo "Results: $PASS passed, $FAIL failed, $SKIP skipped"
[ "$FAIL" -gt 0 ] && exit 1 || exit 0
```

Phase 12 replaces `{{SMOKE_TEST_CHECKS}}` with one `check` call per endpoint from the Discovered Endpoints table. All endpoints get active `check` calls — public endpoints pass `"false"` as the fourth arg, auth-required endpoints pass `"true"`. The function handles the SKIP output internally.

## GraphQL Tech Pack Design

**File:** `technologies/graphql/generic.md`

### Frontmatter

```yaml
---
framework: GraphQL
version: generic
last_updated: 2026-04-14
author: beacon
status: active
---
```

### Section Overview

| Section | Key content |
|---------|-------------|
| 1. Fingerprinting Signals | POST to `/graphql` or `/api/graphql`; `{"query":"..."}` body; `__typename` in responses; `x-hasura-role` header; `__APOLLO_STATE__` global; introspection returns `__schema` |
| 2. Default API Surfaces | Single `/graphql` endpoint; introspection query; WebSocket for subscriptions; Hasura uses `/v1/graphql` |
| 3. JS Globals | `window.__APOLLO_STATE__`, `window.__RELAY_STORE__`, `window.__URQL_DATA__` |
| 4. Auth Patterns | Bearer token in `Authorization`; API key in `x-hasura-admin-secret`; cookie session; JWT claims in custom headers |
| 5. Rate Limiting | Complexity-based (not per-request); depth limiting; alias batching blocks; persisted query enforcement |
| 6. Caching | CDN-unfriendly by default (POST); APQ enables GET + CDN; `@cacheControl` directive (Apollo) |
| 7. Versioning | Schema-first via `@deprecated`; no URL versioning; breaking changes via introspection diff |
| 8. Common Response Patterns | `{"data":{...},"errors":[...]}` — partial success valid; HTTP 200 even on errors |
| 9. Probe Checklist | 8 items: introspection enabled?, mutations available?, subscription endpoint?, auth bypass via unauthenticated introspection?, field-level permissions?, batch query support?, persisted queries enforced?, error messages leak schema? |
| 10. Gotchas | HTTP 200 ≠ success (always check `errors`); introspection often disabled in production; aliases can batch identical queries and bypass rate limits; subscriptions need WebSocket |

# reframe v0.1.0 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. During execution, use the **`plugin-dev`** skills (`plugin-structure`, `skill-development`, `command-development`) for conventions and the **`skill-creator`** skill to author/eval the SKILL.md. Validate with the **`plugin-dev:plugin-validator`** and **`plugin-dev:skill-reviewer`** agents.

**Goal:** Build the `reframe` Claude Code plugin — given an existing site's URL, it extracts purpose/content/IA, critiques it against category best-practice, and emits a paste-ready Claude Design brief + supporting analysis under `docs/redesign/{slug}/`.

**Architecture:** A single prompt-driven skill (`site-redesign`) running a 9-phase, coverage-first pipeline (render → verify → infer), backed by local **category packs** (design best-practice files) and output **templates**. Mirrors Beacon's plugin structure and its session-brief→flush + tool-availability patterns, but contains none of Beacon's API/security scope. Reference: the design spec at `docs/plugins/reframe/designs/2026-06-21-reframe-v0.1.0-design.md`.

**Tech Stack:** Markdown skill + references + templates + category packs; `curl`/Jina/Firecrawl/WebFetch for static fetch, Chrome DevTools MCP for render + screenshots; no compiled code, no runtime dependencies. "Tests" = `plugin-validator` (structure), `skill-creator` evals (process invariants), `skill-reviewer` (description), and a live smoke run.

## Global Constraints

- **Plugin name:** `reframe` (folder `plugins/reframe/`, command namespace `reframe:`). Verbatim.
- **Output dir:** `docs/redesign/{site-slug}/`; slug = strip `www.`, then `example.com → example-com`.
- **Output files (exact names):** `INDEX.md`, `brief.md`, `run-sheet.md`, `content-inventory.md`, `ia-map.md`, `current-critique.md`, and a working `.crawl/` dir.
- **SKILL.md budget:** keep under ~15 KB; push detail to `references/`. Nine lean H2 sections, one per phase. No Beacon-style phase-number bloat (no 6b/8.5 inserts).
- **v1 category packs:** ship exactly four — `generic.md`, `local-service.md`, `saas-marketing.md`, `ecommerce.md` — plus `_TEMPLATE.md`. The remaining six taxonomy entries (`portfolio-personal`, `editorial-blog`, `nonprofit`, `restaurant-hospitality`, `corporate-brochure`, `education-course`, `events`) are a tracked v1.1 follow-up; detection falls back to `generic.md` until they exist. (YAGNI cut from spec §8 — flagged.)
- **WAF fallback chain (trimmed from Beacon):** Firecrawl → Jina → browser-fetch only. No Scrapfly/Spider/DataDome.
- **Degradation philosophy:** unlike Beacon, reframe almost always emits *something* + an assumptions/coverage note; it does NOT hard-stop on partial coverage.
- **Author/version metadata:** match sibling plugins — author `Georgios Pilitsoglou` / `https://pilitsoglou.com`, repo `https://github.com/neotherapper/claude-plugins`, license MIT, version `0.1.0`.

- [ ] **Step 2: Create `crawl-and-coverage.md`** documenting phases 2–4 detail:
  - **Structure discovery:** fetch `robots.txt`, `sitemap.xml` (follow sitemap-index), and homepage nav links; enumerate ALL URLs; cluster by path template via prefix patterns (`/`, `/blog/*`, `/services/*`, `/products/*`, `/about`, `/contact`, legal).
  - **Render gate:** after fetching homepage, count visible text chars + nav links in the returned body. If body text < 200 visible chars OR 0 nav links → log `[RENDER-ESCALATED]` and re-fetch via Chrome DevTools MCP (`navigate_page` + `take_snapshot`/`evaluate_script` to get `document.body.innerText`). Provide the exact Chrome MCP call sequence for both namespaces.
  - **Content-sufficiency gate:** if after render the site still has < 2 unique headings AND < 150 words of non-nav prose → log `[GREENFIELD-MODE]`, stop inferring, tell the user it is not a redesign target, and write only `INDEX.md` with the finding.
  - **Coverage manifest:** record reachable vs gated/blocked (401/403/challenge) URLs; emit `[COVERAGE-PARTIAL:gated]` when applicable.
  - **Crawl budget:** sample 1–2 pages per template cluster; floor = homepage + primary nav targets; bound total on a clean-markdown char budget (default 60,000 chars) — stop when hit even if clusters remain, and log `[SAMPLED:n-templates]`. One screenshot per sampled template (homepage gets above-fold + full-page) via Chrome MCP `take_screenshot`; if Chrome MCP unavailable, log `[TOOL-UNAVAILABLE:chrome-mcp]` and proceed text-only with a noted visual gap.

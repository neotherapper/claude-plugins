## Phase 11 Detailed Design

### Phase 11a — Tool Detection + Auth Setup

**Capability matrix:**

| Tool | JS globals | Network capture | Auth session |
|------|-----------|-----------------|--------------|
| Chrome DevTools MCP (auto-connect) | ✅ | ✅ | ✅ inherits user's Chrome sessions |
| Chrome DevTools MCP (new-instance) | ✅ | ✅ | ❌ needs login |
| cmux browser | ✅ `eval` | ✅ `network requests` | ❌ needs login, but `state save/load` for persistence |
| Neither | — | — | → `[PHASE-11-SKIPPED]` |

**Detection logic (in order):**

```
1. Chrome DevTools MCP present?
   → Check if mcp__chrome-devtools__list_pages is callable
   YES:
     → Call list_pages
     → Returns pages with real URLs? → auto-connect mode → log [CHROME-MODE:auto-connect]
     → Returns empty/blank?          → new-instance mode → log [CHROME-MODE:new-instance]
   NO:
     → Check cmux: [ -n "$CMUX_SURFACE_ID" ] || which cmux
     YES → cmux mode
     NO  → log [PHASE-11-SKIPPED] → skip to Phase 12

2. Auth setup (new-instance or cmux only):
   → Check for .beacon/auth-state.json
     EXISTS → offer: "Load saved auth state? (y/n)"
     LOAD  → cmux: state load .beacon/auth-state.json
             Chrome MCP: note: auto-connect is the recommended path for auth
   → No saved state → offer three options:
     A) "Open browser on {site}, log in, then press Enter to continue"
        → navigate to site, pause, wait for user confirmation
        → on continue: cmux: state save .beacon/auth-state.json
     B) "Skip auth — unauthenticated only"
        → log [PHASE-11-UNAUTH] → proceed to 11b
     C) "Connect to your Chrome (auto-connect setup)"
        → print platform-specific instructions:
          macOS:  open -a "Google Chrome" --args --remote-debugging-port=9222
          Linux:  google-chrome --remote-debugging-port=9222
          Windows: Start-Process chrome -ArgumentList "--remote-debugging-port=9222"
        → instruct user to update MCP config wsEndpoint: "http://localhost:9222"
        → wait for confirmation → re-check list_pages
```

### Phase 11b — Browse Plan Execution

Execute each URL from Phase 10's browse plan (up to 10 URLs, in priority order).

**Per-URL loop — Chrome DevTools MCP:**

```
select_page(page_id)                             ← select the created page
navigate_page(url, type="url", timeout=10000)
evaluate_script(() => document.readyState)       ← poll until "complete"
  → retry up to 3× with 2s delay if not complete

// JS globals (from tech pack Section 3 + catch-all)
evaluate_script(() => {
  const globals = {};
  // Framework-specific globals from tech pack Section 3:
  ['__NEXT_DATA__','__NUXT__','Shopify','__remixContext','__initialData__'].forEach(k => {
    if (window[k]) globals[k] = window[k];
  });
  // Catch-all: any __ prefixed globals
  Object.keys(window).filter(k => k.startsWith('__')).forEach(k => {
    if (!globals[k]) globals[k] = typeof window[k];
  });
  return JSON.stringify(globals);
})

// Network capture (API calls only)
list_network_requests({resourceTypes: ["xhr", "fetch"]})
  → filter client-side: keep only URLs containing target domain
  → for each matching request:
    get_network_request(reqid)   ← captures response body

take_snapshot()                  ← accessibility tree for auth-gate detection
```

**Per-URL loop — cmux browser:**

```bash
cmux browser surface:N goto {url}
cmux browser surface:N wait --load-state complete

# JS globals
cmux browser surface:N eval "JSON.stringify((() => {
  const g = {};
  ['__NEXT_DATA__','__NUXT__','Shopify','__remixContext'].forEach(k => {
    if (window[k]) g[k] = window[k];
  });
  Object.keys(window).filter(k => k.startsWith('__')).forEach(k => { if (!g[k]) g[k] = typeof window[k]; });
  return g;
})()"

# Network capture
cmux browser surface:N network requests

# Snapshot
cmux browser surface:N snapshot --compact
```

**Session brief append per URL:**

```markdown
### Browse: {url}
- Globals: {key}: {value summary}, ...
- API calls observed: {METHOD} {path} ({status}), ...
- Auth-gated: YES/NO
- Snapshot: [compact a11y tree excerpt]
```

### Phase 11c — HAR Reconstruction

The `.beacon/` directory is created in Phase 1 (Scaffold) as a session working directory alongside `docs/research/{site-slug}/`. It is gitignored and not part of the persistent research output.

Save raw captures:
- Chrome MCP: write `list_network_requests` + `get_network_request` results to `.beacon/chrome-requests.json`
- cmux: write `network requests` output to `.beacon/cmux-requests.json`

Run `har-reconstruct.py`:

```bash
python3 scripts/core/har-reconstruct.py \
  --input .beacon/chrome-requests.json \   # or cmux-requests.json
  --output .beacon/capture.har \
  --domain {target-domain}
```

**Script auto-detects format** by presence of `reqid` field (Chrome MCP) vs cmux schema.

**Minimum HAR 1.2 output per entry:**

```json
{
  "startedDateTime": "2026-04-13T10:00:00.000Z",
  "time": 100,
  "cache": {},
  "timings": {"send": 0, "wait": 100, "receive": 0},
  "request": {
    "method": "GET", "url": "https://example.com/api/v1/users",
    "httpVersion": "HTTP/1.1", "headers": [], "cookies": [],
    "queryString": [], "headersSize": -1, "bodySize": -1
  },
  "response": {
    "status": 200, "statusText": "OK",
    "httpVersion": "HTTP/1.1", "headers": [], "cookies": [],
    "redirectURL": "", "headersSize": -1, "bodySize": -1,
    "content": {
      "size": 0,
      "mimeType": "application/json",
      "text": "{\"id\":1}"    ← raw JSON string, NOT base64
    }
  }
}
```

### Phase 11d — OpenAPI Generation + Merge

```bash
npx har-to-openapi .beacon/capture.har \
  --include-domains {site-domain} \
  --format yaml \
  --attempt-to-parameterize-url \
  --infer-parameter-types \
  --guess-authentication-headers \
  --drop-paths-without-successful-response \
  -o docs/research/{site-slug}/specs/{site-slug}.openapi.yaml
```

**Passive spec merge** (if Phase 8 found `specs/{site-slug}.openapi-passive.yaml`):
- Read both specs
- Add endpoints from passive spec not present in observed spec
  - Mark added endpoints with: `x-beacon-source: passive`
- Flag conflicts: `x-beacon-note: "passive spec claims 403, observed 200"`
- Write single merged `specs/{site-slug}.openapi.yaml`
- Remove `openapi-passive.yaml`

**If har-to-openapi not installed and npx unavailable:**
- Log `[OPENAPI-SKIPPED:har-to-openapi-unavailable]`
- Keep `.beacon/capture.har` for manual processing
- Note in INDEX.md

## har-reconstruct.py Design

**Input:** JSON file from either Chrome DevTools MCP or cmux `network requests`

**Format auto-detection:**
- Chrome MCP: entries have `reqid` (number) field
- cmux: different schema (to be confirmed from actual cmux output format)

**Filters applied:**
- Keep only entries matching `--domain` argument
- Skip: analytics domains (google-analytics.com, gtm.js, mixpanel.com, hotjar.com)
- Skip: CDN static assets (images, fonts, .woff, .png, .jpg, .css unless API-like)
- Keep: content-type application/json, text/plain, application/x-www-form-urlencoded

**Output:** valid HAR 1.2 JSON file at `--output` path

## tool-availability.md Corrections

### Chrome DevTools MCP — corrected execution pattern

**What's wrong now:**
- Uses `navigate_page(url, page_id)` — `page_id` doesn't exist
- Uses `wait_for("networkidle", 5000)` — `wait_for` is text-presence only, not networkidle
- Uses `list_network_requests({url_filter: "/api/"})` — no `url_filter` parameter

**Corrected pattern:**
```
1. new_page
2. select_page(page_id) if needed
3. navigate_page(url, type="url", timeout=10000)
4. evaluate_script(() => document.readyState) — poll for "complete"
5. evaluate_script(() => ...) — JS globals
6. list_network_requests({resourceTypes: ["xhr","fetch"]}) — filter URLs client-side
7. get_network_request(reqid) — response bodies
8. take_snapshot()
```

### cmux browser — network requests added

Add to cmux section:
```bash
# Network request capture
cmux browser surface:N network requests        ← list all captured requests
cmux browser surface:N network route "*/api/*" --body '{"mock":true}'  ← intercept/mock
```

## Success Criteria

| Scenario | Expected outcome |
|----------|-----------------|
| Chrome MCP in auto-connect mode, site has JS API | Phase 11 runs fully; `specs/{site}.openapi.yaml` written with inferred schemas |
| Chrome MCP in new-instance mode, user skips auth | Phase 11 runs unauthenticated; spec written for public endpoints only |
| cmux available, no Chrome MCP | Phase 11 runs via cmux; network capture via `network requests`; spec written |
| cmux + auth state file exists | Phase 11 offers to load state; runs authenticated |
| Neither tool available | `[PHASE-11-SKIPPED]`; Phase 12 writes INDEX.md without spec reference |
| har-to-openapi unavailable | `[OPENAPI-SKIPPED:har-to-openapi-unavailable]`; `.beacon/capture.har` preserved |
| Phase 8 found passive OpenAPI spec | Passive and observed specs merged; single `{site}.openapi.yaml` output |

# Strapi 5.x — Tech Pack

## 1. Fingerprinting Signals

| Signal | Type | Value | Confidence |
|--------|------|-------|------------|
| `X-Powered-By: Strapi` response header | HTTP header | `Strapi` | Definitive |
| `X-Strapi-Signature` response header | HTTP header | present | High |
| `/admin/` returns Strapi admin SPA | HTTP probe | React SPA with `strapi` in bundle | High |
| `/api/` returns JSON with `data`/`meta` envelope | HTTP probe | `{"data":...,"meta":...}` | High |
| `_strapi` in JS bundle content | JS content | `_strapi` | High |
| Wappalyzer result | MCP | `"Strapi"` | Definitive |

**Version extraction:**
```bash
curl -sI {site}/api/ | grep -i "x-powered-by"
# Or from admin bundle:
curl -s {site}/admin/ | grep -oP '"strapiVersion":"[^"]+'
```

## 2. Default API Surfaces

| Endpoint | Method | Auth | Notes |
|----------|--------|------|-------|
| `/api/{content-type}` | GET | Public/JWT | List entries (plural kebab-case of content type name) |
| `/api/{content-type}/{id}` | GET | Public/JWT | Single entry |
| `/api/{content-type}` | POST | JWT | Create entry |
| `/api/{content-type}/{id}` | PUT | JWT | Update entry |
| `/api/{content-type}/{id}` | DELETE | JWT | Delete entry |
| `/api/auth/local` | POST | None | Local auth login → returns JWT |
| `/api/auth/local/register` | POST | None | User registration |
| `/api/users/me` | GET | JWT | Current user profile |
| `/graphql` | POST | Public/JWT | GraphQL endpoint (if plugin installed) |
| `/admin/` | GET | Admin session | Admin panel SPA |
| `/_health` | GET | None | Health check |

## 3. Config / Constants Locations

| Location | What's there | How to access |
|----------|-------------|---------------|
| `/api/{content-type}?populate=*` | All populated relation fields + component schemas | Authenticated GET |
| `window.__STRAPI__` | Strapi admin client config | JS eval in admin panel |
| `/admin/main.[hash].js` | Admin panel bundle with content type definitions | JS download + parse |
| Strapi environment `.env` | DB connection, JWT secret, API keys | Source only |
| `/api/content-type-builder/` | Full schema definitions (dev mode only) | Admin authenticated GET |

## 4. Auth Patterns

| Pattern | Location | Notes |
|---------|----------|-------|
| JWT Bearer | `Authorization: Bearer {token}` | Token from `POST /api/auth/local` |
| API Token | `Authorization: Bearer {static_token}` | Created in admin panel → Settings → API Tokens |
| Admin session | `strapi-admin` cookie | For admin panel only |

**JWT acquisition:**
```bash
curl -s -X POST {site}/api/auth/local \
  -H "Content-Type: application/json" \
  -d '{"identifier":"user@example.com","password":"password"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin).get('jwt',''))"
```

**Public vs authenticated content:** In Strapi 5.x, each content type has per-role permissions. A content type may be fully public (no auth), public read/private write, or fully private. Always probe without a token first.

## 5. JS Bundle Patterns

| Path | Content |
|------|---------|
| `/admin/main.[hash].js` | Admin panel SPA (large — contains content type definitions) |
| `/admin/vendor.[hash].js` | Vendor bundle (React, etc.) |
| `/_next/static/` or `/assets/` | Custom frontend if present |

**Extract admin bundle filename:**
```bash
curl -s {site}/admin/ | grep -oP 'main\.[a-f0-9]+\.js'
```

## 6. Source Map Patterns

Strapi admin panel: source maps stripped in production builds.
- Check `/admin/main.[hash].js.map` — usually 404
- Development mode (`strapi develop`) includes source maps with original TypeScript paths
- Custom plugin bundles may include maps if developer forgot to strip them

## 7. Common Plugins & Extensions

| Plugin | API it adds | Detection signal |
|--------|------------|-----------------|
| GraphQL plugin | `POST /graphql` | `graphql` in `/admin/` bundle |
| Users & Permissions (built-in) | `/api/auth/local`, `/api/users/me` | Always present |

| Endpoint | Data | Notes |
|----------|------|-------|
| `/api/{content-type}?populate=*` | Full entry with all relations populated | Use `populate=*` in Strapi 5 |
| `/api/{content-type}?pagination[pageSize]=100` | Bulk fetch up to 100 entries | Default page size is 25 |
| `/api/users/me` | Current user fields | Reveals user model shape |

## 9. Probe Checklist

- [ ] `GET /_health` — confirm Strapi is running; check `X-Powered-By: Strapi` header
- [ ] `GET /api/` — probe API root (may return 404 or list of routes depending on config)
- [ ] `POST /api/auth/local` with dummy creds — confirm auth endpoint exists; observe error shape
- [ ] List known content types: try `/api/articles`, `/api/posts`, `/api/products`, `/api/pages` (common defaults)
- [ ] `GET /api/{found-type}?populate=*` — full entry with relations; extract schema from first result
- [ ] `POST /graphql` with `{"query":"{__schema{types{name}}}"}` — check if GraphQL plugin is active
- [ ] `GET /admin/` — confirm admin panel loads; extract JS bundle filename
- [ ] Download admin bundle and grep for content type names: `grep -oP '"(singularName|pluralName)":"[^"]+'`
- [ ] `GET /api/upload/files` — check if uploaded files are listable (public upload endpoint)
- [ ] Check response headers for `X-Strapi-Signature` on all API responses
- [ ] Enumerate plugin routes — check `/api/` response for non-core plugins (Sentry, i18n, GraphQL)
- [ ] `GET /api/components` — discover Component/Block structures in Strapi v5
- [ ] Check for Strapi v5 changes — compare `/api/` response structure vs v4 (content-type structure changed)
- [ ] `GET /admin/api/roles` — check if admin API is accessible (may require auth)
- [ ] Look for custom content types — grep bundles for `strapi::` service patterns

## 10. Gotchas

- **Content type names are pluralised kebab-case.** A content type named `BlogPost` becomes `/api/blog-posts`. If probing doesn't find it, try singular (`/api/blog-post`) and camelCase variants.
- **Permissions are per-role per-content-type.** A 403 on one endpoint doesn't mean all are protected — probe each content type independently.
- **`populate=*` changed in Strapi v5.** In v4, `populate=*` populated one level. In v5, use `populate[relation][populate]=*` for deep population. Start with `populate=*` and expand as needed.
- **Admin panel is a separate auth context.** Admin users and API users are different auth systems. `POST /api/auth/local` authenticates API users; admin panel has its own login at `/admin/`.
- **GraphQL introspection may be disabled.** `"introspection": false` in the GraphQL plugin config blocks schema discovery. A 400 with "introspection is disabled" is a meaningful finding.
- **Uploads may be public.** The Upload plugin stores files in `/uploads/` (local) or an S3 bucket. Check if `/uploads/` is directly accessible — it often is and contains user-submitted files.
- **Draft vs published state.** Strapi 5 has Draft & Publish. Unauthenticated API only returns published entries. To see drafts, use JWT auth and add `?status=draft`.

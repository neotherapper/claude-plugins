# Shopify 2024-10 — Tech Pack

## 1. Fingerprinting Signals

| Signal | Type | Value | Confidence |
|--------|------|-------|------------|
| `cdn.shopify.com` in HTML | HTML source | CDN hostname present in `<script src>` or `<link href>` | Definitive |
| `window.Shopify` JS object | JS global | Object present in page source | Definitive |
| `shopify-analytics.js` loaded | Script tag | Script src contains `shopify-analytics` | Definitive |
| `myshopify.com` in page source | HTML source | Original store URL embedded in HTML | High |
| `shopify_pay_redirect` cookie | Cookie | Set on checkout initiation | High |
| `_shopify_y` cookie | Cookie | Shopify analytics visitor cookie | High |
| `_shopify_sa_t` cookie | Cookie | Shopify analytics session cookie | High |
| `window.Shopify.shop` | JS global | Contains `{store}.myshopify.com` domain | Definitive |
| `window.Shopify.theme` | JS global | Theme ID, name, role, theme_store_id | High |
| `window.Shopify.apiFeatures` | JS global | Enabled API features array | High |

**Extract version and shop data from HTML:**

```bash
# Extract myshopify domain and Shopify object data from HTML:
curl -s https://STORE_DOMAIN/ | grep -o 'window\.Shopify\s*=\s*{[^<]*' | head -c 500

# Check the Storefront API version in response headers:
curl -I https://STORE_DOMAIN/api/2024-10/graphql.json
```

## 4. Auth Patterns

| Pattern | Location | Notes |
|---------|----------|-------|
| `X-Shopify-Storefront-Access-Token: TOKEN` | HTTP request header | Public token for Storefront API; intentionally embedded in theme for client-side use |
| `Authorization: Bearer ACCESS_TOKEN` | HTTP request header | Admin API via OAuth; obtained through OAuth 2.0 flow |
| `X-Shopify-Access-Token: PRIVATE_TOKEN` | HTTP request header | Admin API private app token; set in Shopify Partner Dashboard |
| `_shopify_s` session cookie | Cookie jar | Customer session authentication |
| Customer access token from `/account/login.json` | POST response body | Used for customer-authenticated Storefront API calls |
| `_shopify_checkout_token` cookie | Cookie jar | Checkout session binding |

**Storefront API token acquisition — search theme HTML and JS:**

```bash
# Search for storefrontAccessToken in HTML source:
curl -s https://STORE_DOMAIN/ | grep -o '"storefrontAccessToken":"[^"]*"' | head -1

# Search for token in script tags (single-quote variant):
curl -s https://STORE_DOMAIN/ | grep -o 'storefrontAccessToken[^,}]*' | head -5

# Search theme JS assets for token:
curl -s https://STORE_DOMAIN/ | grep -o 'cdn\.shopify\.com/s/files/[^"]*\.js' | head -5
```

Note: The Storefront API token is NOT a secret. It is intentionally public and embedded in theme source for client-side use. Finding it is expected behavior, not a misconfiguration.

## 5. JS Bundle Patterns

| Path | Content |
|------|---------|
| `https://cdn.shopify.com/s/files/1/STORE_ID/assets/NAME.js` | Theme JS assets hosted on Shopify CDN |
| `https://cdn.shopify.com/shopifycloud/shopify/assets/shopify_pay*.js` | Shopify Pay / Shop Pay JS |
| `https://cdn.shopify.com/shopifycloud/checkout-web/assets/*.js` | Checkout UI extension JS (Shopify Plus) |
| `https://STORE_DOMAIN/cdn/shop/t/THEME_ID/assets/*.js` | Alternative CDN path for theme assets |
| `/_next/static/chunks/*.js` | Hydrogen (headless React) — Next.js or Remix build artifacts |
| `/build/client/*.js` | Hydrogen with Vite build artifacts |

Extract actual asset URLs from HTML `<script src>` tags — the store ID in CDN paths is store-specific and must be read from the live page. Shopify Liquid stores have no traditional webpack build by default; all JS is CDN-hosted.

## 6. Source Map Patterns

CDN-hosted theme JS for standard Liquid/Storefront stores typically does **not** include source maps. Theme developers rarely publish source maps to Shopify's CDN.

**Check for source maps on CDN assets:**

```bash
# Probe for .map file next to a known CDN script:
# First extract a real script URL from page source:
SCRIPT_URL=$(curl -s https://STORE_DOMAIN/ | grep -o 'https://cdn\.shopify\.com/s/files/[^"]*\.js' | head -1)
# Then probe the .map file:
curl -I "${SCRIPT_URL}.map"
```

**Hydrogen stores (React/Remix headless):**

```bash
# Hydrogen may expose source maps if the build did not disable them:
curl -I https://STORE_DOMAIN/_next/static/chunks/pages/index.js.map
# Check X-SourceMap or SourceMap response headers on JS assets:
curl -I https://STORE_DOMAIN/build/client/root.js
```

Source maps are most likely absent on standard Liquid stores and may or may not be present on Hydrogen stores depending on deployment configuration.

## 9. Probe Checklist

- [ ] `GET /` — check HTML source for `window.Shopify` object and `cdn.shopify.com` in script tags to confirm Shopify fingerprint
- [ ] Extract `window.Shopify` object from HTML — capture `shop`, `theme.id`, `theme.name`, `theme.role`, `apiFeatures` for store metadata
- [ ] `GET /cart.js` — always accessible without auth; confirms Shopify store and returns current cart JSON schema
- [ ] `GET /products.json?limit=10` — enumerate public product catalog; note total count from `X-Total-Count` or pagination
- [ ] `GET /collections.json?limit=10` — enumerate public collections
- [ ] `GET /sitemap.xml` — parse full site structure: all product handles, collection handles, page slugs, blog posts
- [ ] `GET /products/FIRST_HANDLE.json` — retrieve full product schema including variants, options, images, tags, metafields (public)
- [ ] `GET /search.json?q=test` — confirm search endpoint active; note result count for store size estimation
- [ ] Search HTML source for Storefront API token — `grep -o '"storefrontAccessToken":"[^"]*"'` in page source
- [ ] If Storefront token found: `POST /api/2024-10/graphql.json` with `X-Shopify-Storefront-Access-Token` header — test GraphQL introspection or basic product query
- [ ] `GET /account/login` — determine if customer accounts are enabled (200 = enabled, 404 = disabled by merchant)
- [ ] Scan HTML `<script src>` tags for third-party app signals — check for Klaviyo, Recharge, Yotpo, Gorgias, Attentive, LoyaltyLion scripts

## 10. Gotchas

- **Storefront GraphQL vs legacy REST:** Shopify's Storefront API is GraphQL-first in version 2024-10. The legacy REST endpoints (`/products.json`) remain public but the GraphQL Storefront API provides far richer data — metafields, product options, detailed variant pricing, and more. If you found a Storefront access token, prefer GraphQL over REST for data collection.

- **Storefront token is intentionally public:** The `X-Shopify-Storefront-Access-Token` is NOT a secret. Shopify intentionally embeds it in theme JS for client-side use. Finding it in the HTML source is expected behavior and by design — do not treat it as a misconfiguration or vulnerability.

- **Admin API is entirely separate:** The Admin API (`/admin/api/2024-10/`) requires OAuth or a private app token. The public Storefront token grants zero access to admin endpoints. Do not confuse the two; they are different credential systems with different scopes and security models.

- **Quarterly API versioning:** Shopify releases API versions quarterly (`2024-01`, `2024-04`, `2024-07`, `2024-10`). A store may support multiple versions simultaneously. Always probe the latest version first, then fall back to older versions if the endpoint returns a version-not-supported error.

- **Headless Shopify (Hydrogen) is a different fingerprint:** Hydrogen stores are React/Remix apps that use the Storefront API under the hood. Standard Liquid fingerprinting signals (`window.Shopify`, CDN-hosted Liquid assets) may be absent or minimal. Detect Hydrogen separately by looking for React, Vite, or Remix chunk patterns (`/_next/`, `/build/client/`), plus Shopify API calls in network traffic.

- **Rate limiting on `/products.json`:** The public `/products.json` endpoint is rate-limited, especially on high-traffic stores. Use `limit=10` when probing, not `limit=250`. Back off on `429` responses and do not enumerate large catalogs aggressively.

- **Custom domains hide `myshopify.com`:** Merchants use custom domains (e.g., `store.example.com`) and the `myshopify.com` subdomain may not appear in the HTML. Rely on `cdn.shopify.com`, `window.Shopify`, and `shopify-analytics.js` as primary fingerprinting signals, not the myshopify domain.

# Webflow — Tech Pack

Webflow is a visual no-code website builder that exports to static HTML/CSS/JS hosting. It offers CMS, eCommerce, Forms, and Membership features. Sites are typically hosted on Webflow's infrastructure (`*.webflow.io`) or exported to external hosting.

## 1. Fingerprinting Signals

| Signal | Type | Value | Confidence |
|--------|------|-------|------------|
| `content="Webflow"` | HTML meta | exact string in `<head>` | Definitive |
| `X-Webflow-Site` header | HTTP header | site ID or name | Definitive |
| `X-Webflow-Version` header | HTTP header | Webflow version | Definitive |
| `/js/webflow.js` | URL path | exact path in HTML | Definitive |
| `__wf_global` JS global | JavaScript | global object in page | High |
| `wf-style` CSS class | HTML class | presence in `<body>` | High |
| Wappalyzer result | MCP | `"Webflow"` | Definitive |

**Version extraction:**
```bash
# X-Webflow-Version header often contains version info
curl -sI {site} | grep -i "x-webflow-version"

# Generator meta may contain version
curl -s {site} | grep -oP 'content="Webflow[^"]*"'
```

## 2. Default API Surfaces

Webflow has **no public REST API** by default. API access requires:
- Webflow CMS API (requires site API key from Webflow dashboard)
- Webflow Forms API (POST to form endpoint)
- Webflow Membership API (user auth endpoints)

### Form Submissions (no auth for public forms)

| Endpoint | Method | Notes |
|----------|--------|-------|
| `/{form-path}` | POST | Form submission endpoint; requires hidden `name` field |
| `/.wf_forms/` | GET | Webflow form submission API prefix |

**Form discovery:** Forms in Webflow have hidden inputs:
```html
<input type="hidden" name="_wf_namespace" value="abc123">
<input type="hidden" name="_wf submission" value="...">
```

### Webflow CMS API (API key required)

Base URL: `https://api.webflow.com/v1`

| Endpoint | Method | Auth | Notes |
|----------|--------|------|-------|
| `/sites/{site_id}` | GET | API Key | Site info |
| `/sites/{site_id}/collections` | GET | API Key | List CMS collections |
| `/collections/{collection_id}/items` | GET | API Key | Get collection items |
| `/collections/{collection_id}/items/{item_id}` | GET | API Key | Get single item |
| `/sites/{site_id}/ecomm/products` | GET | API Key | List eCommerce products |
| `/sites/{site_id}/ecomm/orders` | GET | API Key | List orders (requires orders permission) |

**API key location:** Webflow Dashboard → Site Settings → API → API Key

### Webflow Membership API (requires user auth)

| Endpoint | Method | Auth | Notes |
|----------|--------|------|-------|
| `/sites/{site_id}/users` | GET/POST | JWT | User management |
| `/sites/{site_id}/login` | POST | None | User login |
| `/sites/{site_id}/signup` | POST | None | User registration |
| `/sites/{site_id}/reset-password` | POST | None | Password reset |

## 3. Config / Constants Locations

| Location | What's there | How to access |
|----------|-------------|---------------|
| `/.wf/` | Internal Webflow paths | Typically blocked |
| `/sitemap.xml` | Auto-generated by Webflow | Public GET |
| `/robots.txt` | Webflow default or custom | Public GET |
| Form HTML source | `name="_wf_*"` hidden fields | Page scrape |
| JSON-LD in page | Schema.org types + Webflow-specific | Page scrape |
| `window.__WF__` | Global with site config | JS eval |

## 4. Auth Patterns

| Pattern | Location | Notes |
|---------|----------|-------|
| None (public site) | N/A | Static pages require no auth |
| Webflow Forms | `POST /{form-id}` | Hidden token in form; submit to Webflow |
| Membership JWT | `Authorization: Bearer {token}` | If site has Webflow Membership enabled |
| API Key | `Authorization: Bearer {api-key}` | For CMS/eCommerce API access |

**Form submission without API key:**
```bash
# Extract form endpoint and submission token from page
curl -s {site} | grep -oP 'action="[^"]+"' | head -1
curl -s {site} | grep -oP 'name="_wf[^"]+" value="[^"]+"' | head -5
```

## 5. JS Bundle Patterns

| Path | Content |
|------|---------|
| `/js/webflow.js` | Core Webflow runtime |
| `/_static/..` | Webflow-hosted static assets |
| `/_js/..` | Custom code injected in site |
| `/.webflow/..` | Webflow designer assets |

**Custom code:** Webflow allows custom code in:
- `<head>` code injection
- `<footer>` code injection
- Individual element custom attributes

## 6. Source Map Patterns

Webflow does **not expose source maps** for its core JS. Custom code injected by users is not minified and may contain:
- Analytics tracking codes
- Third-party widget initializations
- Custom API endpoint references

## 7. Common Plugins & Extensions

Webflow integrations are configured in the Webflow dashboard, not as installable plugins:

| Integration | Detection signal |
|-------------|------------------|
| Google Analytics | `gtag(`, `ga(` in page source |
| Google Tag Manager | `GTM-` container ID |
| Zapier | Form submission → Zapier webhook (in form settings) |
| Memberstack | `memberstack` global, `/memberstack` API calls |
| Webflow CMS | JSON-LD with `@type: "Article"` or collection items |
| Webflow eCommerce | Product schema, `/products` or `/cart` paths |
| Hotjar | `_hjSettings` in page source |
| Intercom | Intercom widget script |

## 9. Probe Checklist

Run these in order. Record result (✓ 200 / ✗ 403 / – 404) for each.

- [ ] `GET {site}` — grep for `content="Webflow"` and `X-Webflow-Site` header
- [ ] `HEAD {site}` — capture `X-Webflow-*` headers for site ID and version
- [ ] `GET {site}/robots.txt` — check for sitemap URL
- [ ] `GET {site}/sitemap.xml` — enumerate all pages
- [ ] `GET {site}` — grep for `/js/webflow.js` path
- [ ] `GET {site}` — grep for `__wf_global` or `wf-style` class
- [ ] `GET {site}` — scan for JSON-LD structured data (`application/ld+json`)
- [ ] `GET {site}` — look for form elements with `name="_wf_` hidden fields
- [ ] `GET {site}/products` — check for eCommerce (if applicable)
- [ ] `GET {site}/cart` — check for shopping cart
- [ ] `GET {site}` — grep for GTM, GA, Intercom, Hotjar scripts
- [ ] `GET {site}` — extract any custom `<script>` tags in head/footer

## 10. Gotchas

- **No public API without API key.** Webflow CMS and eCommerce APIs require an API key from the Webflow dashboard. There is no "guest" or unauthenticated API access.

- **Forms submit to Webflow servers, not the site.** Public forms POST to `submit.webflow.com` or similar. The form endpoint includes a namespace and submission token.

- **Static export removes CMS/eCommerce functionality.** Sites exported via Webflow's "Export" button lose all dynamic CMS/eCommerce features. Detect export sites by checking for static assets or missing `/sitemap.xml` patterns.

- **Membership requires Webflow hosting.** Webflow Membership only works on sites hosted on Webflow's infrastructure (`*.webflow.io` or custom domain with Webflow hosting). Exported sites cannot have member login.

- **Form spam protection varies.** Webflow provides basic CAPTCHA on some forms, but premium sites may use hCaptcha or reCAPTCHA v3. Form submission may fail from automated tools.

- **Webflow CDN serves assets from `d1j8upqvs7n9l5.cloudfront.net`.** Check for assets on this CDN for additional site identification.

- **CMS collections have predictable URL structures.** Collection item URLs follow `/collection-slug/item-slug/`. Collection pages follow `/collection-slug/`.

## 13. Cross-Cutting OSINT Patterns

### Favicon Hashing

Identify technology stack by hashing favicon and searching Shodan/Censys for same stack:

```bash
curl -s "https://{domain}/favicon.ico" | python3 -c "
import sys, mmh3
data = sys.stdin.buffer.read()
try: print('Favicon hash:', mmh3.hash(data))
except: print('Install mmh3: pip install mmh3')
"

# Search Shodan: icon_hash:{hash}
```

**What it reveals:** Hidden subdomains running same framework stack.

### Source Map Discovery

Webflow does not expose source maps, but custom code injections may be readable:

```bash
curl -s "https://{domain}/" | grep -oP '<script[^>]*>.*?</script>' | grep -v "Webflow"
```

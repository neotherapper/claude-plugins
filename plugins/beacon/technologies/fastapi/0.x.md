# FastAPI 0.x — Tech Pack

## 1. Fingerprinting Signals

| Signal | Type | Value | Confidence |
|--------|------|-------|------------|
| `/openapi.json` returns JSON with `openapi:` field | HTTP probe | `"openapi": "3.x.x"` | Definitive |
| `/docs` returns Swagger UI HTML | HTTP probe | `SwaggerUIBundle` in body | High |
| `/redoc` returns ReDoc HTML | HTTP probe | `redoc` in body | High |
| `server: uvicorn` response header | HTTP header | `uvicorn` | High |
| `x-process-time` response header | HTTP header | float string | Medium |
| Wappalyzer result | MCP | `"FastAPI"` | Definitive |

**Version extraction:**
```bash
# From OpenAPI spec info block
curl -s {site}/openapi.json | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('info',{}).get('version',''))"
# FastAPI version itself is rarely exposed; identify from openapi.json presence
```

## 2. Default API Surfaces

| Endpoint | Method | Auth | Notes |
|----------|--------|------|-------|
| `/openapi.json` | GET | None | Full OpenAPI 3.x spec — download immediately |
| `/docs` | GET | None | Swagger UI — interactive explorer |
| `/redoc` | GET | None | ReDoc UI — alternative docs |
| `/api/v1/` | GET | Varies | Common prefix (not enforced by framework) |
| `/health` | GET | None | Common health check (not built-in) |

## 3. Config / Constants Locations

| Location | What's there | How to access |
|----------|-------------|---------------|
| `/openapi.json` | All routes, schemas, tags, security definitions | Public GET |
| `/openapi.json` → `info.title` | App name | JSON parse |
| `/openapi.json` → `components.schemas` | All Pydantic model schemas | JSON parse |
| `/openapi.json` → `components.securitySchemes` | Auth mechanisms declared | JSON parse |
| Python `settings` / `.env` | DB URLs, secret keys | Source only |

## 4. Auth Patterns

| Pattern | Location | Notes |
|---------|----------|-------|
| OAuth2 Bearer (JWT) | `Authorization: Bearer {token}` header | Most common; declare via `OAuth2PasswordBearer` |
| API Key header | `X-API-Key: {key}` | Declared via `APIKeyHeader` dependency |
| API Key query param | `?api_key={key}` | Declared via `APIKeyQuery` dependency |
| HTTP Basic | `Authorization: Basic {b64}` | Declared via `HTTPBasic` dependency |
| Cookie auth | `session` cookie | Via `APIKeyCookie` dependency |

**Token acquisition:**
If OAuth2: `POST /token` (or app-specific path) with `grant_type=password&username=x&password=y`
Check `/openapi.json` → `paths` for any endpoint with `security` and `requestBody` containing `OAuth2PasswordRequestForm`.

## 5. JS Bundle Patterns

FastAPI serves no built-in frontend JS. Only relevant if the app bundles a frontend:

| Path | Content |
|------|---------|
| `/static/*.js` | App-specific static files (if `StaticFiles` mounted) |
| `/assets/*.js` | Common alternative mount point |

Swagger UI and ReDoc are served from CDN by default; no local bundles.

## 6. Source Map Patterns

FastAPI has no source maps (Python backend). If a frontend is co-deployed:
- Check `/static/*.js.map` — only present if custom build pipeline includes them.
- Source maps reveal frontend structure but not Python API routes.

## 8. Known Public Data

| Endpoint | Data | Notes |
|----------|------|-------|
| `/openapi.json` | Complete API schema | Always probe first — it gives everything |
| `/openapi.json` → `paths` | All routes with methods, params, request/response schemas | Parse with `jq .paths` |
| `/openapi.json` → `components.schemas` | All data models | Reveals field names and types |
| `/openapi.json` → `tags` | Logical groupings of endpoints | Map to API surface areas |

## 9. Probe Checklist

Run these in order. Record result (✓ 200 / ✗ 403 / – 404) for each.

- [ ] `GET /openapi.json` — download and save as `specs/{slug}.openapi-passive.yaml` (convert from JSON if needed)
- [ ] `GET /docs` — confirm Swagger UI loads; note if protected
- [ ] `GET /redoc` — confirm ReDoc loads
- [ ] `GET /health` — common health endpoint (not always present)
- [ ] `GET /api/v1/` — probe common API version prefix
- [ ] `GET /api/v2/` — probe v2 prefix (some apps version their APIs)
- [ ] Check `/openapi.json` → `paths` for any `POST` endpoints requiring auth — note them for Phase 11
- [ ] Check `/openapi.json` → `components.securitySchemes` — extract token endpoint URL
- [ ] `POST {token_endpoint}` with dummy creds — observe error shape (reveals auth mechanism)
- [ ] Check for `/graphql` (Strawberry or Ariadne integration)

## 10. Gotchas

- **OpenAPI is the primary signal.** FastAPI auto-generates `/openapi.json` for every app. Always download it in Phase 8 — it eliminates the need for most Phase 5 probing.
- **Docs can be disabled.** Production apps often set `docs_url=None` and `redoc_url=None`. A 404 on `/docs` does not mean the app isn't FastAPI — check headers and `/openapi.json` separately.
- **No standard route prefix.** Unlike Django REST Framework's router or Laravel's `/api/`, FastAPI has no default prefix. Routes can be at root `/` or any prefix — read `/openapi.json` paths.
- **Multiple routers.** Large apps use `APIRouter` with different prefixes included at different paths. `/openapi.json` always shows the full merged picture.
- **Background tasks don't appear in OpenAPI.** Celery and similar async jobs have no HTTP surface; don't expect them in the spec.
- **CORS headers are explicit.** If `CORSMiddleware` is present, its `allow_origins` setting reveals expected frontend origins.

## 11. GitHub Code Search Patterns

Use these queries on GitHub to find custom endpoints, plugin code, and configuration examples for this framework.

### Framework-Specific Queries

| Search Query | What it finds |
|--------------|---------------|
| `"@app.get(" language:python` | FastAPI GET endpoint definitions |
| `"@app.post(" language:python` | FastAPI POST endpoint definitions |
| `"def " language:python path:app/` | FastAPI route handler functions |
| `"FastAPI(" language:python` | FastAPI application instantiation |


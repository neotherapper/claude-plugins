|--------|------|-------|------------|
| `wp-content/` in HTML | HTML substring | `wp-content/` | High |

| `wordpress_logged_in_*` | Cookie | prefix match | High |
| Wappalyzer result | MCP | `"WordPress"` | Definitive |

**Version extraction:**
```bash
curl -s {site} | grep -oP 'content="WordPress \K[\d.]+'
# Also try:
curl -s {site}/wp-json/ | python3 -c "import sys,json; print(json.load(sys.stdin).get('name',''))"
```

## 3. Config / Constants Locations

| Location | What's there | How to access |
|----------|-------------|---------------|
| `wp-config.php` | DB creds, secret keys, table prefix | Source only |
| `/wp-json/` root response | REST namespace list, site name, URL | Public GET |
| `window.wpApiSettings` | REST nonce, root URL | JS eval in browser |
| `window.wp.apiFetch` | REST fetch wrapper | JS eval |
| `data-nonce` HTML attributes | Per-form nonces | HTML scrape |
| `/wp-json/wp/v2/settings` | Site title, tagline, URL | Admin auth required |

## 4. Auth Patterns

| Pattern | Location | Notes |
|---------|----------|-------|
| WP REST nonce (`X-WP-Nonce` header) | `wp_create_nonce('wp_rest')` injected in page | Valid ~24h; fetch from `window.wpApiSettings.nonce` |
| Application passwords | Basic auth header | WP 5.6+; `user:app-password` base64 encoded |
| Cookie auth | `wordpress_logged_in_*` | Session cookie; browser-only |
| JWT (plugin-dependent) | Bearer token | Requires JWT plugin |

**Nonce acquisition (authenticated operations):**
The REST nonce is injected into `window.wpApiSettings.nonce` when WordPress loads
`wp-api.min.js`. Also available at `window.wp.apiFetch.nonceMiddleware.nonce`.
Fetch via browser JS eval or grep inline scripts for `"nonce":"`.

## 6. Source Map Patterns

WordPress core JS files do NOT have source maps in production.
Custom themes compiled with webpack/vite/rollup MAY have source maps.
Check: `{theme-bundle}.js.map` ŌĆö if 200, recover original source paths.

## 9. Probe Checklist

Run these in order. Record result (Ō£ō 200 / Ō£Ś 403 / ŌĆō 404) for each.

- [ ] `GET /wp-json/` ŌĆö REST API root (lists namespaces; confirms REST is enabled)
- [ ] `GET /wp-json/wp/v2/` ŌĆö Core namespace (lists all routes with methods)
- [ ] `GET /wp-json/wp/v2/posts?per_page=1` ŌĆö Confirm public posts accessible
- [ ] `GET /wp-json/wp/v2/categories?per_page=100` ŌĆö Full category list with IDs
- [ ] `GET /wp-json/wp/v2/tags?per_page=100` ŌĆö Full tag list with IDs
- [ ] `GET /wp-json/wp/v2/users?per_page=10` ŌĆö User list (may be 403)
- [ ] `GET /wp-json/wp/v2/types` ŌĆö Custom post types
- [ ] `GET /wp-json/wp/v2/taxonomies` ŌĆö Custom taxonomies
- [ ] `GET /wp-json/wp/v2/search?search=test` ŌĆö Search endpoint
- [ ] `GET /wp-sitemap.xml` ŌĆö Sitemap (WP 5.5+)
- [ ] `GET /feed/` ŌĆö RSS feed
- [ ] `POST /wp-admin/admin-ajax.php` with `action=heartbeat` ŌĆö AJAX endpoint
- [ ] `GET /wp-json/wc/v3/` ŌĆö WooCommerce (if WC detected)
- [ ] `GET /wp-json/acf/v3/` ŌĆö ACF (if detected)
- [ ] Check `/wp-json/` namespace list for any non-core namespaces
- [ ] `GET /wp-json/` ŌĆö enumerate ALL namespaces (not just core), check for custom post types and taxonomies
- [ ] `POST /xmlrpc.php` with `system.listMethods` ŌĆö enumerate XML-RPC methods available
- [ ] Check for WordPress multisite: `GET /sites/` or look for `blog_id` in cookies/source
- [ ] `GET /wp-json/wp/v2/settings` ŌĆö attempt to access site settings (requires auth, confirms auth setup)
- [ ] Check for REST API namespace plugins: grep `/wp-json/` response for non-core namespaces like `acf/v3`, `woocommerce`, `yoast/v1`
- [ ] `POST /wp-admin/admin-ajax.php` with `action=query_themes` ŌĆö test AJAX endpoint with various actions
- [ ] `GET /wp-json/wp/v2/users?per_page=100` ŌĆö attempt user enumeration (may be blocked by security plugins)

## 10. Gotchas

- **REST API can be completely disabled.** Plugins like "Disable REST API" return 401/403
  on all `/wp-json/` requests. If this happens: fall back to AJAX endpoint + XML-RPC.
- **XML-RPC is often active** even on secured sites. Try `POST /xmlrpc.php` with
  `<methodCall><methodName>system.listMethods</methodName></methodCall>`.
- **Nonce stability.** The REST nonce changes daily (based on WP secret key + salt),
  NOT per-request. Fetch once at session start and reuse.
- **User enumeration blocking.** Many security plugins block `/wp-json/wp/v2/users`.
  A 403 here is expected; note it but don't treat as a site-wide auth issue.
- **Namespace sprawl.** Large sites have 20+ namespaces. Always read the full namespace
  list from `/wp-json/` and probe each non-standard one.
- **WooCommerce API versioning.** WC uses `/wc/v1/`, `/wc/v2/`, `/wc/v3/`. Always test
  the latest (`v3`) first; older versions may still be active.
- **REST API authentication changes in WP 6.x.** Application Passwords (introduced 5.6)
  are the preferred auth method for REST API. Cookie+nonce still works for browser-based calls.

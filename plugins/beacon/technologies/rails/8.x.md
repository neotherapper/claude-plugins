# Rails 8.x — Tech Pack

## 1. Fingerprinting Signals

| Signal | Type | Value | Confidence |
|--------|------|-------|------------|
| `authenticity_token` | HTML form hidden field | `<input type="hidden" name="authenticity_token" ...>` | High — Rails CSRF |
| `_session_id` | Cookie name | `_session_id=...` | High — default Rails session cookie |
| `X-Runtime` | Response header | Decimal seconds, e.g. `0.043721` | High — Rails default |
| `X-Request-Id` | Response header | UUID string | Medium — Rails default |
| `ETag` | Response header | Quoted hash string | Medium — Rails default |
| `"Ruby on Rails"` | Wappalyzer result | Technology fingerprint | High — Wappalyzer detection |
| `X-Powered-By` | Response header | May reveal Rails version | Low — often absent in production |

**Version extraction:**

```bash
# Sometimes reveals version via X-Powered-By (rare in production):
curl -s https://site/ -I | grep X-Powered-By

# Development only — explicit version info page:
curl -s https://site/rails/info/properties
```

Better signals: error pages in development mode, or `GET /rails/info/properties` if accessible (returns 403 in production on a correctly-configured app).

## 3. Config / Constants Locations

| Location | How to access | Contains |
|----------|---------------|----------|
| `config/credentials.yml.enc` | Server-side only; decrypted with `RAILS_MASTER_KEY` | Database passwords, API keys, secrets |
| `config/database.yml` | Server-side only | DB adapter, host, database name, pool size |
| `<meta name="csrf-token">` in `<head>` | `curl -s https://site/ \| grep csrf-token` | Per-session CSRF token |
| `authenticity_token` in forms | `curl -s https://site/users/sign_in \| grep authenticity_token` | Per-session CSRF token (form variant) |
| Asset manifest (Propshaft) | `/assets/manifest-{hash}.json` | Digest map of asset filenames |
| Asset manifest (Sprockets) | `/assets/` directory listing (if indexes enabled) | Fingerprinted JS/CSS paths |
| `<script type="importmap">` | Parsed from any HTML page | All JS module paths (Rails 7-8 default) |

## 4. Auth Patterns

| Pattern | Location | Notes |
|---------|----------|-------|
| CSRF token (form) | `<input type="hidden" name="authenticity_token">` | Must be sent with all non-GET requests |
| CSRF token (header) | `<meta name="csrf-token" content="...">` in `<head>` | Send as `X-CSRF-Token` header |
| Session cookie | `_session_id` cookie (encrypted) | Opaque; acquire via cookie jar |
| Devise session | `/users/sign_in` POST → Set-Cookie | Most common Rails auth gem |
| JWT (knock / jwt_sessions) | `Authorization: Bearer <token>` | Common in API-only Rails apps |
| API key | `X-API-Key` or `Authorization: Token token=<key>` | Custom header, app-defined |

**CSRF token acquisition:**

```bash
# From meta tag (recommended — works on any page load):
CSRF=$(curl -sc /tmp/cookies.txt https://site/ \
  | grep -o 'content="[^"]*" name="csrf-token"' \
  | grep -o 'content="[^"]*"' \
  | cut -d'"' -f2)

# From a sign-in form hidden input:
CSRF=$(curl -s https://site/users/sign_in \
  | grep -o 'value="[^"]*" name="authenticity_token"' \
  | grep -o 'value="[^"]*"' \
  | cut -d'"' -f2)
```

The CSRF token is per-session, not per-request. Fetch it once per session and reuse.

## 5. JS Bundle Patterns

| Path | Content |
|------|---------|
| `/assets/application-{digest}.js` | Main JS bundle (Sprockets or Propshaft) |
| `/assets/{name}-{digest}.js` | Individual asset chunks |
| `app/javascript/controllers/` | Stimulus controllers (Hotwire) |
| `app/javascript/` | Application-level JS source |
| `<script type="importmap">` in HTML | All JS module paths — read to discover entry points |

**Rails 8 uses Propshaft (new) or Sprockets (legacy).** Hashed filenames mean you must extract the real filename from the HTML before fetching:

```bash
# Step 1: Extract the actual hashed filename from the HTML source:
ASSET_URL=$(curl -s https://site/ | grep -o '/assets/application-[a-f0-9]*.js' | head -1)

# Step 2: Fetch the resolved asset:
curl -s "https://site${ASSET_URL}"
```

**Import maps (Rails 7-8 default) expose all JS entry points directly in the HTML:**

```bash
# Extract and pretty-print the import map JSON from the HTML:
curl -s https://site/ | grep -o '"importmap">[^<]*</script>' | python3 -m json.tool
```

## 6. Source Map Patterns

Rails asset pipelines (Propshaft and Sprockets) do not generate source maps by default in production.

- **Propshaft:** No source maps generated by default. Asset files are digest-renamed only.
- **Sprockets:** No source maps in production. Development may expose unminified assets.
- **Vite-Rails (vite_ruby gem):** May generate source maps if `sourcemap: true` is set in `vite.config.ts`.
- **cssbundling-rails / jsbundling-rails:** Depends on bundler config (esbuild, rollup, webpack). Source maps follow bundler settings.

**Check for source maps after resolving the real asset filename:**

```bash
# Step 1: Resolve hashed filename from HTML:
ASSET_URL=$(curl -s https://site/ | grep -o '/assets/application-[a-f0-9]*.js' | head -1)

# Step 2: Probe for a .map file at the same path:
curl -I "https://site${ASSET_URL}.map"
```

Expect `404` in production for Propshaft/Sprockets apps. A `200` indicates a misconfigured or development build was deployed.

## 9. Probe Checklist

- [ ] `curl -I https://site/` — check for `X-Runtime`, `X-Request-Id`, `X-Powered-By` headers
- [ ] `curl -s https://site/ | grep csrf-token` — extract CSRF token from meta tag
- [ ] `GET /rails/info/properties` — 200 means dev/misconfigured (read everything); 403 = production
- [ ] `GET /rails/info/routes` — 200 means dev/misconfigured; full route table exposed
- [ ] `GET /users/sign_in` — Devise present? Note the model name (users, accounts, admins, etc.)
- [ ] `GET /admin/` — ActiveAdmin mounted? Note redirect target for login
- [ ] `GET /sidekiq` — Sidekiq Web UI? 401/403 = exists but protected; 404 = not mounted
- [ ] `curl -s https://site/ | grep -o '<script type="importmap">.*</script>'` — parse import map for all JS module paths
- [ ] `GET /{resource}.json` — probe common REST resources (`/users.json`, `/posts.json`, `/articles.json`)
- [ ] `GET /cable` — Action Cable endpoint present? HTTP returns 426 Upgrade Required (expected)
- [ ] `GET /api/v1/` — API mode routes; try with `Accept: application/json` header
- [ ] `curl -s https://site/ | grep -o 'data-turbo\|<turbo-frame'` — Hotwire/Turbo present?

## 10. Gotchas

- `/rails/info/properties` and `/rails/info/routes` are only accessible in development. A 403 in production is expected and correct. If you get a 200 in production, the app is misconfigured — read every detail before the operator notices.
- The CSRF token is per-session, not per-request. Fetch it once by loading any page with `Set-Cookie` response, store the session cookie, and reuse the token for all non-GET requests in that session.
- Rails format negotiation: the same URL returns HTML or JSON depending on the `Accept` header or a `.json` suffix. Always probe endpoints with `Accept: application/json` to check if a JSON API exists at routes you'd expect to return HTML only.
- Import maps (default in Rails 7-8) expose every JS entry point directly in the HTML `<script type="importmap">` block. Parsing it reveals all application modules and their cache-busted paths — far more useful than guessing asset filenames.
- Devise route prefixes are model-derived: `/users/sign_in` is the default but the model could be `Account`, `Admin`, `Member`, etc. Check HTML links, the import map, or error messages to discover the actual model name before assuming `/users/`.
- Action Cable at `/cable` requires a WebSocket upgrade. `curl` will receive `426 Upgrade Required` — this is expected and confirms the endpoint exists. Do not treat it as a probe failure; note it and use a WS client to interact.
- Hashed asset filenames (Propshaft digests, Sprockets fingerprints) cannot be guessed. Always extract the real filename from an HTML `<script src>` or `<link href>` tag first, then fetch it — never construct asset URLs from patterns alone.

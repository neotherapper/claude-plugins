# nopCommerce 4.x — Tech Pack

nopCommerce is an open-source ASP.NET Core e-commerce platform (NOT ASP.NET WebForms). It runs on
.NET 6+ with Kestrel or IIS, uses Razor Pages and MVC routing, and stores data in SQL Server. Current
stable release is 4.70.x. Popular in Eastern Europe, the Middle East, and mid-market Western European
retail. Do not apply WebForms signals (`__VIEWSTATE`, `ScriptResource.axd`, `.aspx` URLs) here —
none of those exist in nopCommerce.

## 1. Fingerprinting Signals

| Signal | Type | Value | Confidence |
|--------|------|-------|------------|
| `<meta name="generator" content="nopCommerce" />` | HTML meta | exact string in `<head>` | Definitive |
| `.Nop.Authentication` cookie | Cookie | exact name | Definitive |
| `.Nop.Customer` cookie | Cookie | exact name | High |
| `.Nop.Antiforgery` cookie | Cookie | exact name | High (combined with above = Definitive) |
| `X-Powered-By: ASP.NET` header | HTTP header | IIS-hosted only; absent on Kestrel | Low alone |
| `/Admin/` path returns 302 → `/Admin/login` | URL probe | capital-A `Admin`; lowercase `/admin` redirects to same | High |
| `/Themes/{ThemeName}/Content/` CSS paths in HTML | HTML source | e.g. `/Themes/DefaultClean/Content/` | High |
| `/Plugins/` static asset paths in HTML | HTML source | e.g. `/Plugins/Nop.Plugin.*/` | High |
| `Nop.Web` in `Server` header or error pages | HTTP header / HTML | reveals Kestrel hosting | Medium |
| `__RequestVerificationToken` hidden field | HTML field | ASP.NET Core anti-forgery — confirms Core MVC | Low alone |
| Wappalyzer result | MCP | `"nopCommerce"` | Definitive |
| `/sitemap.xml` with nopCommerce route pattern | URL probe | includes `/sitemap-{N}.xml` paged format | Medium |
| `nopCommerce` string in page HTML comments | HTML source | present in some default theme layouts | Medium |

**Version extraction:**
```bash
# Generator meta tag (version only sometimes present)
curl -s {site} | grep -oiP 'content="nopCommerce[^"]*"'

# Admin login page reveals version in page title or JS globals on some installs
curl -s {site}/Admin/login | grep -i "nopcommerce\|version"

# Check /sitemap.xml — paged sitemap is nopCommerce-specific
curl -s {site}/sitemap.xml | head -30

# Cookie inspection — .Nop.* prefix is definitive
curl -sc /tmp/nop_cookies.txt -s {site} -o /dev/null && cat /tmp/nop_cookies.txt | grep -i "\.Nop\."
```

## 2. Default API Surfaces

nopCommerce has no built-in REST API. All REST capability requires an installed API plugin.
Two dominant plugins exist:

**SevenSpikes/api-plugin-for-nopcommerce** (open-source, supports up to 4.20; community forks extend to 4.50):
- OAuth 2.0 Authorization Code grant type
- Base path: `/api/`

**nopCommerce Official Web API plugin** (commercial, supports 4.50–4.80, OpenAPI 3.0):
- Swagger UI at `{site}/api/index.html`
- JWT Bearer token authentication + API key header

**Core store routes** (always present — no plugin required):

| Endpoint | Method | Auth | Notes |
|----------|--------|------|-------|
| `/` | GET | None | Storefront home |
| `/{slug}` | GET | None | Any entity by slug — product, category, manufacturer, or topic page; all resolve at root via `SlugRouteTransformer` |
| `/search` | GET | None | Search results; `?q={term}` |
| `/cart` | GET/POST | Session | Shopping cart |
| `/checkout` | GET | Auth | Checkout — redirects to login if guest not allowed |
| `/login` | GET/POST | None | Customer login |
| `/register` | GET/POST | None | Customer registration |
| `/customer/info` | GET | Auth | Customer account info |
| `/wishlist` | GET | Auth | Wishlist page |
| `/sitemap.xml` | GET | None | XML sitemap index |
| `/sitemap-{N}.xml` | GET | None | Paged sitemap (N = 0, 1, 2…) |
| `/robots.txt` | GET | None | Robots file — lists sitemap URL |
| `/Admin/` | GET | Admin auth | Admin panel root; redirects to `/Admin/login` |
| `/Admin/login` | GET/POST | None | Admin login form |

**REST API endpoints** (Official Web API plugin, if installed):

| Endpoint | Method | Auth | Notes |
|----------|--------|------|-------|
| `/api/index.html` | GET | None | Swagger UI — lists all available endpoints |
| `/api/token` | POST | API key | Obtain JWT access token |
| `/api/products` | GET | Bearer | Product list (paginated) |
| `/api/products/{id}` | GET | Bearer | Single product by ID |
| `/api/categories` | GET | Bearer | Category list |
| `/api/categories/{id}` | GET | Bearer | Single category |
| `/api/customers` | GET | Bearer + Admin | Customer list |
| `/api/customers/{id}` | GET | Bearer + Admin | Single customer |
| `/api/orders` | GET | Bearer + Admin | Order list |
| `/api/orders/{id}` | GET | Bearer + Admin | Single order |
| `/api/shoppingcartitems` | GET/POST | Bearer | Shopping cart items |

**SevenSpikes plugin** (open-source variant, older stores):

| Endpoint | Method | Auth | Notes |
|----------|--------|------|-------|
| `/api/products` | GET | OAuth token | Supports `?fields=id,name,images` |
| `/api/products/count` | GET | OAuth token | Product count only |
| `/api/products/{id}` | GET | OAuth token | Single product |
| `/api/customers` | GET | OAuth token | Customer list |
| `/api/customers/search?query=first_name:john` | GET | OAuth token | Customer search |
| `/api/orders/customer/{customer_id}` | GET | OAuth token | Orders for customer |
| `/api/categories` | GET | OAuth token | Category list |
| `/api/shoppingcartitems` | GET/POST/DELETE | OAuth token | Cart management |

## 3. Config / Constants Locations

| Location | What's there | How to access |
|----------|-------------|---------------|
| `appsettings.json` | DB connection string, data provider type, logging | File access only; IIS blocks direct HTTP access |
| `web.config` | IIS handler config only (minimal in Core) | 403 from IIS — confirms IIS hosting |
| `/Admin/Configuration/Settings/generalsettings` | Store name, URL, defaults | Admin auth required |
| `/Admin/Configuration/Settings/catalogsettings` | Product URL structure type, paging | Admin auth required |
| `window.nopConfig` or inline JS `var localized` | JS globals injected per page | JS eval in browser |
| `__RequestVerificationToken` hidden field | Anti-forgery token per form | HTML scrape — needed for any POST |
| `.Nop.Customer` cookie value | Customer GUID (registered users) | Cookie header inspection |
| `/sitemap.xml` | All public product/category/page slugs | Public GET |
| `/robots.txt` | Disallow paths including `/Admin/`; sitemap link | Public GET |

## 4. Auth Patterns

| Pattern | Location | Notes |
|---------|----------|-------|
| Cookie auth (storefront) | `.Nop.Authentication` cookie | Set on `/login` POST; 1-year expiry for "remember me" |
| Customer GUID | `.Nop.Customer` cookie | Tracks guest and registered customers; 1-year expiry |
| Anti-forgery | `.Nop.Antiforgery` cookie + `__RequestVerificationToken` hidden field | Required for all POST operations on storefront |
| Admin cookie auth | `.Nop.Authentication` (admin role scope) | Admin panel uses same auth system with role check |
| API key (Official Web API) | `X-API-KEY` request header | Per-application key configured in Admin panel |
| JWT Bearer (Official Web API) | `Authorization: Bearer {token}` | Obtained from `/api/token`; expires per plugin config |
| OAuth 2.0 (SevenSpikes plugin) | `Authorization: Bearer {access_token}` | Authorization Code flow; token endpoint at `/oauth/authorize` |

**Acquiring anti-forgery token for storefront POST:**
```bash
# 1. GET the page — extract token and save session cookie
TOKEN=$(curl -sc /tmp/nop_cookies.txt -s {site}/login | \
  grep -oP '__RequestVerificationToken" value="\K[^"]+')

# 2. POST with token in body and session cookie
curl -sb /tmp/nop_cookies.txt -s -X POST {site}/login \
  -d "__RequestVerificationToken=${TOKEN}&Email=user@example.com&Password=secret&RememberMe=false"
```

**Obtaining JWT for Official Web API plugin:**
```bash
# Step 1: get JWT by authenticating with API key
curl -s -X POST {site}/api/token \
  -H "X-API-KEY: {your-api-key}" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin@store.com","password":"adminpass"}'
# Response: {"access_token":"eyJ...","token_type":"Bearer","expires_in":3600}

# Step 2: use JWT for subsequent calls
curl -s {site}/api/products \
  -H "Authorization: Bearer eyJ..."
```

## 5. JS Bundle Patterns

| Path | Content |
|------|---------|
| `/Themes/{ThemeName}/Content/scripts/public.common.min.js` | Core storefront JS — search for AJAX cart/checkout calls |
| `/Themes/{ThemeName}/Content/scripts/public.ajaxcart.min.js` | AJAX add-to-cart handler (default theme) |
| `/Themes/DefaultClean/Content/css/styles.css` | Default theme stylesheet |
| `/Plugins/{PluginSystemName}/Content/` | Plugin-specific static assets |
| `/lib/jquery/jquery.min.js` | jQuery served from wwwroot/lib |
| `/lib/bootstrap/js/bootstrap.bundle.min.js` | Bootstrap |
| Inline `<script>` in page `<head>` | `var localized = {...}` — localised strings and AJAX URLs |

**Extracting custom AJAX endpoints from JS:**
```bash
curl -s {site}/Themes/DefaultClean/Content/scripts/public.common.min.js \
  | grep -oP '"(/[a-z][a-zA-Z/{}?=&]+)"' | sort -u
```

Theme name varies per install (common defaults: `DefaultClean`, `Avenue`). Find it from HTML source:
```bash
curl -s {site} | grep -oP '/Themes/\K[^/]+' | head -1
```

## 6. Source Map Patterns

nopCommerce ships minified JS without source maps in production builds. Custom themes compiled with
webpack or Vite may expose `.map` files if the developer left them in place.

```bash
# Check default theme bundles
curl -s {site}/Themes/DefaultClean/Content/scripts/public.common.min.js -I \
  | grep -i "sourcemappingurl"
curl -o /dev/null -w "%{http_code}" \
  {site}/Themes/DefaultClean/Content/scripts/public.common.min.js.map

# Check plugin JS files
curl -s {site} | grep -oP '/Plugins/[^"]+\.js' | while read p; do
  code=$(curl -o /dev/null -w "%{http_code}" -s "{site}${p}.map")
  echo "$code $p.map"
done
```

A 200 on a `.map` file recovers the original TypeScript/ES6 source paths and may reveal custom
store logic or internal endpoint URLs embedded in source comments.

## 9. Probe Checklist

Run these in order. Record result (✓ 200 / ✗ 403 / – 404) for each.

- [ ] `GET {site}` — grep for `<meta name="generator" content="nopCommerce"` — definitive confirmation
- [ ] `HEAD {site}` — capture `Set-Cookie` for `.Nop.Authentication`, `.Nop.Customer`, `.Nop.Antiforgery`
- [ ] `HEAD {site}` — capture `X-Powered-By`, `Server` headers (IIS vs Kestrel hosting)
- [ ] `GET {site}/robots.txt` — verify `/Admin/` is disallowed; extract sitemap URL
- [ ] `GET {site}/sitemap.xml` — confirm nopCommerce paged sitemap format (`sitemap-{N}.xml` links)
- [ ] `GET {site}/sitemap-0.xml` — harvest all product and category slugs from first page
- [ ] `GET {site}/Admin/` — expect 302 → `/Admin/login` (capital A; lowercase also redirects)
- [ ] `GET {site}/Admin/login` — check page title and HTML for nopCommerce version indicators
- [ ] `GET {site}/search?q=test` — confirm search route format; note response HTML structure
- [ ] `GET {site}/api/index.html` — 200 = Official Web API plugin installed; read Swagger for all endpoints
- [ ] `GET {site}/api/products` — 200 = API open/guest; 401 = API present but auth required; 404 = no plugin
- [ ] `GET {site}/api/categories` — same as above; cross-reference with sitemap slugs
- [ ] `GET {site}/api/products/count` — SevenSpikes variant returns `{"count": N}`; verify plugin type
- [ ] `GET {site}` — grep HTML for `/Themes/{ThemeName}/` — extract theme name for JS path discovery
- [ ] `GET {site}/Themes/{ThemeName}/Content/scripts/public.common.min.js` — grep for AJAX URLs
- [ ] `GET {site}/Themes/{ThemeName}/Content/scripts/public.ajaxcart.min.js` — AJAX cart logic
- [ ] `GET {site}/Themes/{ThemeName}/Content/scripts/public.common.min.js.map` — 200 = source map available
- [ ] `GET {site}/web.config` — expect 403 (confirms IIS; absent on Kestrel = 404)
- [ ] `GET {site}` — grep HTML for `/Plugins/` paths — identify installed plugins by system name
- [ ] `GET {site}/{product-slug}` (from sitemap) — verify slug routing, extract product schema data
- [ ] Check page HTML for `__RequestVerificationToken` hidden field — required for all POST operations

## 10. Gotchas

- **nopCommerce is ASP.NET Core MVC, not WebForms.** There is no `__VIEWSTATE`, no
  `__EVENTVALIDATION`, no `ScriptResource.axd`, no `.aspx` URLs, no `trace.axd`, and no `elmah.axd`.
  Probing for WebForms artifacts will return 404 and tells you nothing about this platform.

- **URL routing is slug-based and prefix-free for all entity types.** Products, categories,
  manufacturers, and CMS topic pages all resolve at root — e.g. `/{slug}` — via the
  `SlugRouteTransformer`. There is no `/product/`, `/category/`, or `/manufacturer/` URL prefix
  in the default configuration. The only way to know whether a slug is a product or a category is
  to fetch the page and inspect the HTML (look for `productTemplate` or `categoryTemplate` CSS
  classes and breadcrumb structure). Use the sitemap to enumerate all slugs, then classify by
  page content. Admins can optionally enable a hierarchical `/category-slug/product-slug` structure
  in Catalog Settings, but this is non-default and changes only product URLs, not category URLs.

- **`/Admin/` uses capital A — it is not `/admin/`.** Both `/admin/` and `/Admin/` resolve to the
  same admin panel in practice (ASP.NET Core routing is case-insensitive by default), but the
  canonical path in nopCommerce source and documentation is `/Admin/`. Security tools may probe
  `/admin` and miss `/Admin/` if case sensitivity is enforced at the load balancer.

- **The API is optional — most nopCommerce stores have no REST API.** The official Web API plugin
  and the SevenSpikes plugin are separate installs. A 404 on `/api/products` is normal; it means
  no API plugin is installed, not that the store is blocked. Fall back to HTML scraping and sitemap
  harvesting when no API is found.

- **Two different API plugins have different authentication schemes.** The SevenSpikes plugin uses
  OAuth 2.0 Authorization Code flow. The Official nopCommerce Web API plugin uses JWT Bearer tokens
  obtained via an API key. They have different token endpoints and different endpoint schemas — do not
  assume one applies when you detect the other. Identify which is installed via `/api/index.html`
  Swagger content or error message format.

- **Anti-forgery tokens are session-bound.** The `__RequestVerificationToken` in every HTML form is
  tied to the `.Nop.Antiforgery` cookie from the same session. You must GET the target form page,
  save the cookie jar, extract the token, then POST — all with the same cookie jar. A token from a
  different session returns HTTP 400 with an anti-forgery validation error.

- **Cookie prefix `.Nop.` is nopCommerce-specific but not version-specific.** The cookie names
  `.Nop.Authentication`, `.Nop.Customer`, `.Nop.Antiforgery` are stable across 4.x and do not
  encode the version. For version information, check the `/Admin/login` page HTML or the Swagger
  UI response body.

- **`X-Powered-By: ASP.NET` only appears on IIS-hosted installs.** Kestrel-hosted nopCommerce
  (Linux containers, Azure App Service without IIS) emits no `X-Powered-By` header. The absence
  of this header does not rule out nopCommerce — always cross-check with the generator meta tag
  and `.Nop.*` cookies.

- **Plugin system names appear in asset URLs.** URLs like `/Plugins/Nop.Plugin.Payments.Stripe/`
  and `/Plugins/Nop.Plugin.Misc.SendinBlue/` in HTML source reveal installed plugins by their
  system names directly. Enumerate all `/Plugins/` paths found in source — each is a separate
  plugin that may add its own admin routes or API surface.

- **The paged sitemap format is nopCommerce-specific.** `/sitemap.xml` is an index that links to
  `/sitemap-0.xml`, `/sitemap-1.xml`, etc. Each page contains up to 1,000 slugs for products,
  categories, manufacturers, topics, and news. This is one of the most reliable non-authenticated
  data sources for discovering full catalog structure.

- **Guest checkout is configurable.** Some nopCommerce stores disable guest checkout, forcing
  registration before purchasing. This affects whether unauthenticated POST to `/checkout` works
  or redirects to `/login`. Always probe `/checkout` with a guest session first.

- **nopAjaxCart is a third-party plugin, not a built-in signal.** The `nopAjaxCart` name refers
  to an unofficial third-party AJAX cart plugin by Nop-Templates.com — it is not part of the
  nopCommerce core and is not present by default. Do not use it as a fingerprinting signal; use
  the `.Nop.*` cookies and generator meta tag instead.

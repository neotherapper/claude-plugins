# Ghost 5.x — Tech Pack

## 1. Fingerprinting Signals

| Signal | Type | Value | Confidence |
|--------|------|-------|------------|
| `Ghost-Version` response header | HTTP header | semver string | Definitive + Version |
| `X-Ghost-Version` response header | HTTP header | semver string | Definitive + Version |
| `ghost/` paths in HTML | HTML substring | `ghost/` | High |
| `/ghost/` admin redirect | HTTP probe | 301/302 to `/ghost/#/signin` | High |
| `<meta name="generator" content="Ghost` | HTML meta | `Ghost` | High |
| Wappalyzer result | MCP | `"Ghost"` | Definitive |

**Version extraction:**
```bash
curl -sI {site} | grep -i "ghost-version"
# Or from Content API:
curl -s "{site}/ghost/api/content/settings/?key={content_api_key}" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('settings',{}).get('version',''))"
```

## 3. Config / Constants Locations

| Location | What's there | How to access |
|----------|-------------|---------------|
| `/ghost/api/content/settings/?key={key}` | Site title, description, timezone, accent colour | Authenticated GET |
| HTML source `<script>` | `ghostApiKey` or `contentApiKey` variable | HTML grep |
| Theme JS bundles | API key and site URL | JS grep |
| `window.__ghost` or `window.ghost` | Client-side config object | JS eval in browser |
| `/ghost/` admin panel | Staff users, integrations, API keys | Authenticated admin session |

## 4. Auth Patterns

| Pattern | Location | Notes |
|---------|----------|-------|
| Content API key | `?key={key}` query param | Read-only public API; key visible in theme source |
| Admin API key (JWT) | `Authorization: Ghost {jwt}` header | JWT signed with Admin API secret; never public |
| Member session | `ghost-members-ssr` cookie | For paid/free member-gated content |
| Staff session | `ghost-admin-api-key` + session | For Ghost Admin panel |

**Content API key extraction:**
```bash
# Often hardcoded in Casper theme or custom theme JS
curl -s {site} | grep -oP '(?<=contentApiKey["\s:=]+)[a-f0-9]{26}'
# Also check /assets/built/*.js
```

## 5. JS Bundle Patterns

| Path | Content |
|------|---------|
| `/assets/built/*.js` | Compiled theme JS (Casper or custom) |
| `/assets/built/*.css` | Compiled theme CSS |
| `/public/cards.min.js` | Ghost card components |
| `/public/member-attribution.min.js` | Member attribution tracking |

**Finding bundle URLs:**
```bash
curl -s {site} | grep -oP '/assets/built/[^"'\'']+\.(js|css)'
```

## 6. Source Map Patterns

Ghost's Casper theme and built themes do not ship source maps in production. Custom themes compiled with webpack may include them — check `/assets/built/*.js.map`.

## 7. Common Plugins & Extensions

Ghost does not have a traditional plugin system; integrations are via Zapier, webhooks, or custom API integrations.

## 9. Probe Checklist

- [ ] `curl -sI {site}` — check for `Ghost-Version` header; extract version
- [ ] `GET /ghost/` — confirm redirect to admin panel (301 to `/ghost/#/signin`)
- [ ] Search HTML source for Content API key pattern (`[a-f0-9]{26}`)
- [ ] `GET /ghost/api/content/settings/?key={found_key}` — verify key works; get site config
- [ ] `GET /ghost/api/content/posts/?key={key}&limit=5` — get recent posts and extract tag/author IDs
- [ ] `GET /ghost/api/content/tags/?key={key}&limit=all` — full tag list for constants.md
- [ ] `GET /ghost/api/content/authors/?key={key}` — author list
- [ ] `GET /ghost/api/content/tiers/?key={key}` — check if paid membership tiers exist
- [ ] `GET /members/api/member/` — check if member session endpoints are active
- [ ] `GET /sitemap.xml` — download sitemap for Phase 2 URL discovery

## 10. Gotchas

- **Content API key is intentionally public.** Unlike the Admin API key, the Content API key is embedded in theme source and is meant to be visible — it only grants read access to published content.
- **Admin API key must never be exposed.** If you find it in frontend source, log it as a critical security finding.
- **Members API is separate from Content API.** `/members/api/` serves authenticated member endpoints (subscription status, comments). It uses `ghost-members-ssr` session cookies, not the Content API key.
- **Posts can be member-gated.** `visibility: "members"` or `visibility: "paid"` posts return 401 to the Content API without a member session.
- **Ghost Headless vs Traditional.** Some Ghost installations use it headless (API only, custom frontend). The Content API is identical but there may be no `/ghost/` admin panel path.
- **Rate limiting.** Ghost Content API rate limits by IP. Space probes ≥1s apart during heavy enumeration.

# Next.js 15.x — Tech Pack

## 1. Fingerprinting Signals

| Signal | Type | Value | Confidence |
|--------|------|-------|------------|
| `_next/static/` in HTML | HTML substring | `_next/static/` | Definitive |
| `__NEXT_DATA__` JSON in page source | HTML script tag | `id="__NEXT_DATA__"` | Definitive |
| `x-powered-by: Next.js` header | HTTP header | `Next.js` | High |
| `next` in `package.json` | File content | `"next":` | High (if accessible) |
| Wappalyzer result | MCP | `"Next.js"` | Definitive |
| `_next/image` endpoint active | HTTP probe | 400/200 response | High |

**Version extraction:**
```bash
curl -s {site} | grep -oP '"version":"\K[\d.]+(?=")' 
# Or from __NEXT_DATA__:
curl -s {site} | grep -oP '"buildId":"\K[^"]+'
```

**Router detection:**
- Pages Router: `__NEXT_DATA__` script tag is present in HTML
- App Router: no `__NEXT_DATA__`; uses RSC payload; `_next/static/` still present

**Build ID extraction:**
```bash
grep -o '"buildId":"[^"]*"' index.html
```

## 3. Config / Constants Locations

| Location | How to access | Contains |
|----------|---------------|----------|
| `window.__NEXT_DATA__` | Browser JS eval | `props`, `pageProps`, `query`, `buildId`, `assetPrefix`, `page` |
| `/_next/static/chunks/main-*.js` | Public HTTP GET | May expose `NEXT_PUBLIC_*` env var names and values |
| `/_next/static/{buildId}/_buildManifest.js` | Public HTTP GET | All client-navigable page paths |
| `/_next/static/{buildId}/_ssgManifest.js` | Public HTTP GET | List of SSG-rendered pages |
| `.next/BUILD_ID` | Not publicly accessible | Build ID string |
| `next.config.js` / `next.config.ts` | Not publicly accessible | Not publicly accessible |

**Extracting `__NEXT_DATA__` in browser:**
```javascript
JSON.parse(document.getElementById('__NEXT_DATA__').textContent)
```

**Extracting from HTML source:**
```bash
curl -s {site} | grep -oP '(?<=<script id="__NEXT_DATA__" type="application/json">)[^<]+'
```

## 4. Auth Patterns

| Pattern | Signals | Discovery |
|---------|---------|-----------|
| NextAuth.js | `/api/auth/session`, `/api/auth/providers`, `/api/auth/csrf` | GET `/api/auth/session` → 200 with JSON |
| Clerk | `/__clerk/` routes, `NEXT_PUBLIC_CLERK_*` vars in bundle | Check bundle for `CLERK` string |
| Raw JWT cookie | `next-auth.session-token` cookie | Check Set-Cookie headers |
| Auth.js v5 | `/api/auth/*` same as NextAuth | Same discovery as NextAuth |
| Server Actions | POST to page URL with `Next-Action` header | Not discoverable from client without RSC payload |

**NextAuth session check:**
```bash
curl -s {site}/api/auth/session
# Returns {} if unauthenticated, or session object if authenticated
```

**NextAuth providers list:**
```bash
curl -s {site}/api/auth/providers
# Returns JSON of configured OAuth providers
```

## 5. JS Bundle Patterns

| Path Pattern | Router | Contents |
|-------------|--------|----------|
| `/_next/static/chunks/pages/*.js` | Pages Router | Per-page JS bundles |
| `/_next/static/chunks/app/*.js` | App Router | Layout and page component chunks |
| `/_next/static/chunks/webpack-*.js` | Both | Webpack runtime |
| `/_next/static/chunks/framework-*.js` | Both | React + React DOM |
| `/_next/static/chunks/main-*.js` | Both | Main entrypoint; may contain `NEXT_PUBLIC_*` vars |
| `/_next/static/{buildId}/_buildManifest.js` | Both | Manifest of all page paths and their chunks |
| `/_next/static/css/*.css` | Both | Extracted CSS modules |

**Scanning for leaked env vars:**
```bash
# Chunk filenames are content-hashed and must be extracted from the HTML source first — glob patterns do not work in URLs.
# Step 1: extract the actual hashed main chunk filename from the HTML
MAIN_CHUNK=$(curl -s {site} | grep -o '_next/static/chunks/main-[^"]*\.js' | head -1)
# Step 2: then fetch it
curl -s {site}/$MAIN_CHUNK | grep -oP 'NEXT_PUBLIC_\w+'
```

**Getting all page paths from manifest:**
```bash
curl -s {site}/_next/static/{buildId}/_buildManifest.js \
  | grep -oP '"[^"]*"(?=:)' | sort -u
```

## 6. Source Map Patterns

| Scenario | Source Maps Available | Path Pattern |
|----------|-----------------------|-------------|
| Default production build | No | N/A |
| `productionBrowserSourceMaps: true` in next.config | Yes | `/_next/static/chunks/*.js.map` |
| Vercel deployment (default) | No | N/A |
| Vercel with explicit opt-in | Yes | `/_next/static/chunks/*.js.map` |
| Development mode (`next dev`) | Yes | Inline or `/_next/static/chunks/*.js.map` |

**Check for source maps:**
```bash
# Extract real chunk filename first:
MAIN_CHUNK=$(curl -s {site} | grep -o '_next/static/chunks/main-[^"]*\.js' | head -1)
curl -I {site}/$MAIN_CHUNK.map
```

**Note:** Production source maps are disabled by default in Next.js. Presence in production is a misconfiguration or intentional opt-in.

## 8. Known Public Data

| Data | Source | How to Extract |
|------|--------|---------------|
| SSR page props | `window.__NEXT_DATA__.props.pageProps` | Inspect in browser console or grep HTML |
| Build ID | `window.__NEXT_DATA__.buildId` | Same as above |
| Current page path | `window.__NEXT_DATA__.page` | Same as above |
| URL query params at render time | `window.__NEXT_DATA__.query` | Same as above |
| All client-side page paths | `/_next/static/{buildId}/_buildManifest.js` | Fetch manifest after extracting buildId |
| SSG page list | `/_next/static/{buildId}/_ssgManifest.js` | Fetch after extracting buildId |
| `NEXT_PUBLIC_*` env var values | `/_next/static/chunks/main-*.js` | Grep bundle for `NEXT_PUBLIC_` prefix |
| OAuth providers configured | `/api/auth/providers` (if NextAuth) | GET request |

**Full data extraction flow:**
```bash
# 1. Get buildId
BUILD_ID=$(curl -s {site} | grep -oP '"buildId":"\K[^"]+')
# 2. Get all pages
curl -s {site}/_next/static/${BUILD_ID}/_buildManifest.js
# 3. Get pageProps from SSR page
curl -s {site} | grep -oP '(?<=<script id="__NEXT_DATA__" type="application/json">)[^<]+'
```

## 9. Probe Checklist

Run these in order. Record result (✓ 200 / ✗ 403 / – 404) for each.

- [ ] `GET /` — check HTML source for `__NEXT_DATA__` (Pages Router) or `_next/static/` (both routers)
- [ ] Check `x-powered-by` response header on `GET /` — should contain `Next.js`
- [ ] Check `x-nextjs-cache` and `x-nextjs-*` response headers for cache status and preview mode signals
- [ ] Extract `__NEXT_DATA__` from HTML: `grep -o '"buildId":"[^"]*"' index.html` to capture Build ID
- [ ] `GET /_next/static/{buildId}/_buildManifest.js` — full page manifest (requires buildId from previous step)
- [ ] `GET /_next/static/{buildId}/_ssgManifest.js` — SSG page list
- [ ] `GET /_next/image?url=/favicon.ico&w=32&q=75` — image optimizer active? (400 = active but bad params, 200 = success)
- [ ] `GET /api/auth/session` — NextAuth present? (200 = active, 404 = not present)
- [ ] `GET /api/auth/providers` — NextAuth provider list (200 = configured, 404 = not present)
- [ ] `GET /api/trpc/` — tRPC present? (200/400 = active, 404 = not present)
- [ ] Grep `/_next/static/chunks/main-*.js` for `NEXT_PUBLIC_` — leaked env var names/values
- [ ] Extract `main-*.js` chunk filename from HTML source → check for `.map` suffix (`curl -I {site}/$MAIN_CHUNK.map`)

## 10. Gotchas

- **App Router vs Pages Router behave very differently for recon.** App Router has no `__NEXT_DATA__` script tag — it uses RSC (React Server Components) payload which is a binary-ish streaming JSON format. Detect App Router by presence of `_next/static/` combined with _absence_ of `__NEXT_DATA__`. Many probes that work on Pages Router will yield nothing on App Router.

- **Build ID changes on every deploy.** Cache it from `__NEXT_DATA__.buildId` (Pages Router) or from any `_next/static/` path in HTML (App Router) early in the session. All `/_next/data/` and `/_next/static/{buildId}/` requests require the current Build ID — stale IDs return 404.

- **Server Actions POST to the page URL — not a separate endpoint.** They look like regular form submissions. The `Next-Action: {hash}` request header identifies them. Server Action hashes are not discoverable from the client side unless you parse the RSC payload or read source. Don't mistake them for REST endpoints.

- **`NEXT_PUBLIC_*` variables are intentionally exposed; non-public env vars are not.** Variables without the `NEXT_PUBLIC_` prefix are server-side only and will never appear in JS bundles. Do not waste time searching bundles for `DATABASE_URL`, `SECRET_KEY`, etc. — they are stripped at build time by design.

- **`/api/` routes are Route Handlers in App Router — not necessarily RESTful.** They may accept any HTTP method and follow no naming convention. Discovery requires reading `_buildManifest.js` or the app source. A 404 on `/api/` itself is normal and not a sign the app has no API routes.

- **Next.js 15 has changed caching defaults.** Unlike Next.js 13/14 where `fetch()` was cached by default, Next.js 15 defaults to no-store. SSR pages may return fresh data on every request. Do not assume `/_next/data/` endpoints return stale cached content.

- **Middleware runs at the edge before the response.** If you get unexpected redirects, rewrites, or authentication challenges, the site likely has a `middleware.ts` file. Middleware is not discoverable from outside — its presence must be inferred from behavior (e.g., 307 redirects to `/login` on protected routes).

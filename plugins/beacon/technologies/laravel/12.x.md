# Laravel 12.x — Tech Pack

## 1. Fingerprinting Signals

| Signal | Type | Value | Confidence |
|--------|------|-------|------------|
| `laravel_session` cookie | Cookie | `laravel_session` | Definitive |
| `X-Powered-By: PHP` header | HTTP header | `PHP` | Medium (not Laravel-specific) |
| `XSRF-TOKEN` cookie | Cookie | `XSRF-TOKEN` | High |
| `<meta name="csrf-token"` in HTML | HTML meta | `csrf-token` content | High |
| `/vendor/` paths in HTML assets | HTML substring | `/vendor/` | Medium |
| Wappalyzer result | MCP | `"Laravel"` | Definitive |

**Version extraction:**
```bash
# From X-Powered-By or framework headers (often disabled in prod)
curl -sI {site} | grep -i "x-laravel"
# Laravel version rarely exposed; check composer.json if accessible
curl -s {site}/composer.json 2>/dev/null | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('require',{}).get('laravel/framework',''))" 2>/dev/null
```

## 4. Auth Patterns

| Pattern | Location | Notes |
|---------|----------|-------|
| Sanctum SPA auth | `GET /sanctum/csrf-cookie` → `POST /login` | Sets `laravel_session` + `XSRF-TOKEN` cookies |
| Sanctum API tokens | `Authorization: Bearer {token}` | Token issued from `/api/user/token` or custom endpoint |
| Passport JWT | `Authorization: Bearer {jwt}` | OAuth2 server; token from `/oauth/token` |
| Fortify | `POST /login` with `_token` field | Returns redirect + session cookie |
| Session cookie | `laravel_session` cookie | For web routes (not API routes) |

**Sanctum SPA auth flow:**
```bash
# Step 1: get CSRF cookie
curl -c cookies.txt {site}/sanctum/csrf-cookie
# Step 2: login with CSRF token from cookie
XSRF=$(grep XSRF cookies.txt | awk '{print $7}' | python3 -c "import sys,urllib.parse; print(urllib.parse.unquote(sys.stdin.read().strip()))")
curl -b cookies.txt -c cookies.txt -X POST {site}/login \
  -H "X-XSRF-TOKEN: $XSRF" \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"password"}'
```

## 5. JS Bundle Patterns

| Path | Content |
|------|---------|
| `/js/app.js` | Main compiled JS (Laravel Mix or Vite) |
| `/build/assets/*.js` | Vite-compiled chunks (Laravel 12 default) |
| `/build/manifest.json` | Vite asset manifest — maps logical names to hashed filenames |
| `/js/bootstrap.js` | Axios setup with CSRF token injection |
| `/vendor/laravel/sanctum/public/` | Sanctum JS (rare) |

**Finding bundle URLs via manifest:**
```bash
curl -s {site}/build/manifest.json | python3 -c "import sys,json; [print(v['file']) for v in json.load(sys.stdin).values() if 'file' in v]"
```

## 6. Source Map Patterns

- Laravel Mix: check `/js/app.js.map` (present in dev, stripped in prod by default)
- Vite: check `/build/assets/[name]-[hash].js.map` — strip `sourceMappingURL` comment from bundle to find map path
- Source maps reveal original Vue/React component paths and API call sites

- [ ] `GET /api/` — probe for API index or 404 (reveals if API routes exist)
- [ ] `GET /api/user` — test Sanctum auth (expect 401 if unauthenticated)
- [ ] `GET /sanctum/csrf-cookie` — confirm Sanctum is present; sets XSRF-TOKEN cookie
- [ ] `GET /login` — capture CSRF token from HTML for session auth
- [ ] `GET /_debugbar/open` — check for Laravel Debugbar (dev environments)
- [ ] `GET /telescope/` — check for Telescope (staging environments)
- [ ] `GET /horizon/` — check for Horizon queue dashboard
- [ ] `GET /build/manifest.json` — download Vite manifest for asset path discovery
- [ ] `GET /storage/` — probe for exposed storage (should return 403/404)
- [ ] Check response headers for `X-RateLimit-Limit` and `X-RateLimit-Remaining`
- [ ] `GET /oauth/token` — check if Passport OAuth2 server is active (expect 405 or JSON error)
- [ ] `POST /livewire/message/{component}` — test Livewire AJAX endpoints (extract component names from HTML)
- [ ] Read `vite-manifest.json` from HTML source — check `/build/manifest.json` or `/hot` for Vite dev server
- [ ] `POST /broadcasting/auth` — Laravel Echo/WebSocket auth endpoint (Pusher/Ably)
- [ ] `GET /sanctum/csrf-cookie` — get CSRF cookie for SPA authentication
- [ ] Check for `.env` exposure — `GET /.env` or `GET /.env.backup` (critical misconfiguration)
- [ ] `GET /api/user` — common Laravel API route for authenticated user info
- [ ] Check for Octane patterns — look for Swoole/RoadRunner headers (`x-octane-`)

## 10. Gotchas

- **CSRF required for all state-changing requests.** Laravel rejects POST/PUT/PATCH/DELETE without `X-XSRF-TOKEN` header (or `_token` form field). Always fetch the CSRF cookie first.
- **API routes vs web routes.** `routes/api.php` routes get the `/api/` prefix and use token auth. `routes/web.php` routes use session auth and CSRF. They are separate middlewares.
- **Sanctum SPA vs API tokens.** SPA auth uses cookie sessions (fetch CSRF cookie first). API token auth uses Bearer headers. The same `/api/` routes can accept both.
- **Debugbar/Telescope in production is a critical finding.** If `/telescope/` or `/_debugbar/` responds, log it prominently — these tools expose full request/response history including credentials.
- **`/storage/` exposure.** If `public/storage` is symlinked and directory listing is on, user uploads are browsable. Always probe.
- **Rate limiting headers.** Laravel's Throttle middleware adds `X-RateLimit-*` headers. Observe them to avoid hitting limits during recon.

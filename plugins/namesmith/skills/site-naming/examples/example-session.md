# Example Session: Developer Productivity SaaS

This example shows a complete namesmith run — from project description through brand interview, Wave 1 results, Wave 2 refinement, and final `names.md`.

---

## User prompt

> I'm building a code review automation tool for engineering teams. It integrates with GitHub and Slack, reads PR diffs, and leaves intelligent comments. I need a domain for it.

---

## Step 1: Project file detection

No README, package.json, or pyproject.toml found in current directory. Proceeding to brand interview.

---

## Step 2: Personal brand detection

No personal brand signals detected. Proceeding to standard interview.

---

## Step 3: Brand Interview

**Q1 — What are you building?**

> Code review automation tool for engineering teams. Integrates with GitHub and Slack, reads PR diffs, leaves intelligent comments.

---

**Q2 — How should the name feel?**

> b) Authoritative/benchmark — Credible, established, serious (Bloomberg, Stripe, Notion, Atlassian)

---

**Q3 — What direction for the name itself?**

> a) Functional — The name says what it does or who it's for

---

**Q4 — TLD budget preference?**

> b) Balanced — Mix of .com, .io, .dev, .app

---

**Q5 — Name length preference?**

> a) Short & punchy — 6 characters or fewer

---

**Q6 — Hard constraints?**

> No hyphens. Prefer .io or .com. Nothing that sounds like a developer joke.

---

## Brand profile locked

```
Brand profile locked:
- Building: Code review automation for engineering teams (GitHub + Slack integration)
- Tone: B — Authoritative/benchmark
- Direction: A — Functional
- Mode: B — Balanced
- Length: A — Short & punchy (≤6 chars)
- Constraints: No hyphens. Prefer .io or .com. No developer jokes.
```

**Applied weighting rules:**
- Q2=B + Q3=A → Descriptive ×2, Compound/Mashup ×2
- Q5=A → Short & Punchy ×2; cap names at 6 chars where possible
